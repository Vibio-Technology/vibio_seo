# 能力路由与降级方案

先选择所需能力，再选择可用提供商。诚实开展工作从不以某个特定提供商为前提。

| 能力 | 首选证据/提供商 | 降级方案 | 不可用时绝不能做的事 |
|---|---|---|---|
| 实时 SERP | DataForSEO，或可指定目标地区的浏览器/搜索能力 | 用户导出，或人工抽取目标市场样本 | 虚构排名、结果类型、PAA 或竞争对手覆盖情况 |
| 关键词需求 | GSC、Ads 搜索词、Keyword Planner、DataForSEO | RFQ/CRM、本地语言竞争对手用语，以自动补全/PAA 作为措辞证据 | 虚构搜索量/KD，或将 Ads 竞争程度称为自然 KD |
| 网站抓取 | Firecrawl 或抓取工具导出 | 通过 sitemap 发现页面，再进行范围明确的 curl/浏览器/本地抓取 | 从一个页面推断已覆盖全站 |
| Search Console | `seo-google` 或直接访问/导出的 GSC 数据 | 索取导出；检查可索引性和渲染产物 | 无数据时声称索引、点击、查询或排名位置表现 |
| 分析平台/CRM | GA4、BigQuery、CRM 导出 | 定义埋点方案和决策字段 | 声称实现查询级收入归因 |
| 渲染验证 | 本地构建 + 渲染后 HTML/浏览器 | 检查源码，并明确标记为未经渲染验证 | 未经检查就声称标签或内容渲染正确 |
| 结构化数据 | Rich Results Test、Schema Markup Validator、解析器 | 解析 JSON-LD，并与最新官方功能文档对照 | 把存在 Schema 当作排名证明 |
| 外链 | DataForSEO/Ahrefs/Semrush/数据导出 | 搜索、引荐日志、人工核验的来源页面 | 虚构权威指标或有害链接结论 |
| 性能 | CrUX/PageSpeed/Lighthouse | 实测请求/渲染耗时 | 将实验室评分当作真实用户现场性能 |
| AI 可见性 | Search Console 当前实际提供的 AI 功能报告/控制、平台人工证据 | 若无独立报告，保留普通 Web 口径并做带日期的非穷尽人工抽样 | 从普通 Web 数据反推 AI 专属表现，或将 `llms.txt` / crawler token 当 Google AI 排名代理 |
| 付费搜索验证 | Google Ads Planner、搜索词、付费与自然搜索报告、实验 | 小规模人工研究集或用户导出 | 将广告支出视为自然排名因素 |

## 能力检查

工作流开始时：

1. 列出需要证据支持的决策。
2. 检查可用提供商、凭据、数据导出和本地产物。
3. 选择当前可用的最强来源。
4. 说明降级方案以及仍然无法得知的事项。
5. 继续开展范围明确的工作，不得虚构精确度。

外部专业工具名称及其降级方案在 `vibio.manifest.yaml` 中声明，供仓库校验。即使未安装任何外部工具，运行时指令仍必须可执行。
