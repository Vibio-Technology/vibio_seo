# 多 Agent 对抗性内容审查

目标是让不同审查者独立寻找失败原因，而不是用一个任意总分证明内容会排名。事实、来源、意图、原创价值、真实经验与可发布性是硬闸门；分数只用于比较版本和定位弱项。

## 为什么分开审查

单一审稿人容易受大纲和写作过程影响，也难同时深入验证来源、SERP 意图、技术事实和语言。独立审查能减少确认偏误，但“盲”不等于剥夺完成任务所需的证据。

不同维度收到不同最小证据包；审查者之间不共享结论。

## 五个审查角色

### 1. 意图与页型审查

输入：目标市场/语言、目标 query/词族、当前 SERP 页型摘要、草稿。

检查：

- 页面类型和主任务是否匹配目标 SERP 与买家阶段。
- 标题/开头承诺与正文是否一致。
- 是否混入不属于同一意图的主题或商业路径。
- 哪些内容应拆页、合并或删除。

硬失败：明显页型错配，或页面无法完成目标任务。

### 2. 非同质化价值审查

输入：SERP coverage-gap matrix、一手素材清单、草稿。

检查：

- 草稿是否真正使用一手经验、原创数据、独特决策框架或真实媒体。
- 所谓“信息增益”是否只是换词、加长或重新排版。
- 哪些贡献足以让目标用户优先选择该页面。
- 是否存在无证据的“我们实测/客户发现”。

硬失败：没有可具体指出的非同质化贡献，或虚构一手经验。

### 3. 事实与来源审查

输入：草稿、证据表、允许使用的项目事实。

逐项建立 claim -> source 映射，检查：

- URL 是否可访问，原文是否真的支持论断。
- 数字、单位、年份、样本、市场和适用条件是否一致。
- 是否引用原始来源而非二手转述。
- 时效性强的事实是否已现场核验。
- 无来源的数字、幽灵统计或错误推断。

硬失败：关键事实错误、来源不存在/不支持论断、虚构数字或引用。

### 4. 目标读者任务与可用性审查

输入：目标读者/任务、草稿（不提供写作过程）。

检查：

- 读者能否理解条件、比较选项、核验事实并采取下一步。
- 结构是否由任务驱动；表格、列表、图像是否真的有帮助。
- 是否缺少必要定义、风险、失败模式、单位或商业条件。
- CTA 是否与买家阶段匹配。
- 移动端或 CMS 发布后是否仍可用。

不要强制 answer-first、固定段落长度、每个 H2 首句、FAQ 或表格；Google 不要求 AI 专用格式。

硬失败：重要步骤/条件缺失，或交付结构无法实际发布/使用。

### 5. 语言与编辑审查

输入：目标语言、品牌语气、草稿。

检查：

- 表达是否自然、具体、专业且符合目标市场。
- 是否存在套话、模糊归因、夸张承诺、重复或机械关键词。
- 术语、单位、大小写和命名是否一致。
- 技术表格/列表是否承载真实信息，不能因“去 AI”被误删。
- 翻译是否属于本地化表达，而非直译。

硬失败：语言严重妨碍理解、出现虚假承诺或不符合目标市场的关键术语。

## 判定规则

每个审查者输出：

```json
{
  "dimension": "...",
  "hard_gate_passed": true,
  "confidence": "high|medium|low",
  "verified_evidence": ["..."],
  "blocking_findings": ["..."],
  "important_improvements": ["..."],
  "optional_edits": ["..."],
  "score_for_version_comparison": 0
}
```

`score_for_version_comparison` 可用 0-20，但不能作为排名预测或交付的唯一依据。

交付条件：

1. 所有硬闸门通过。
2. Blocking finding 已修复或用证据证明不适用。
3. 关键 claim 均能映射到有效来源或允许使用的一方事实。
4. 草稿至少包含可具体指出的非同质化贡献。
5. 发布所需 metadata、来源、内链与责任信息完整。

任何高分都不能抵消硬失败。

## 迭代流程

1. 并行启动独立审查（能力不足时按角色逐个运行，但不要把前一个结论告诉后一个）。
2. 合并 blocking findings，按硬闸门和业务影响去重排序。
3. 先修事实/来源，再修意图/原创价值，最后修结构和语言。
4. 只重跑受改动影响的角色；重大重写则全部重跑。
5. 如果问题来自缺少一手材料、客户确认或有效来源，停止改写并明确请求证据，不能靠继续生成文本解决。

不设置为了达分而无限循环的固定轮数。连续两轮没有实质改进，或剩余问题依赖外部材料时，标记 blocker 并停止。

## 防止审查器误判

- 不给“信息增益审查”一篇孤立草稿却要求它比较 SERP；必须提供 gap matrix。
- 不给“来源审查”只看正文；必须提供证据表并打开来源核对。
- 不把 E-E-A-T 当作 schema/作者框的机械得分。
- 不把固定字数、段落长度、标题字符数、关键词密度或 AI 摘引格式作为质量标准。
- 不因语言简洁就删除必要条件、规格和例外。
- 不把 reviewer 主观偏好升级成强制修改；每项意见要说明影响哪个用户任务或证据关卡。

