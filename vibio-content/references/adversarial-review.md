# 多 Agent 盲审循环 — 95 分闸门

这是 CONTENT 引擎 Step 6 的升级版：**不再是一个人对着 checklist 挑刺，而是派出 5 个独立 Agent，每人只看稿子、按不同维度盲打分。平均分不到 95 就改，改完再送审，直到过关。**

## 为什么这么设计

单人审稿有致命弱点：
- **上下文污染**：审稿人知道关键词、看过竞品、参与过大纲讨论 → 已经有预期，发现不了「陌生人读不懂」的问题
- **单一视角**：一个人同时审视 EEAT、信息增益、可读性、去AI、技术准确性 → 每项都看，每项都不深
- **没有硬闸门**：checklist 审稿是「勾完了事」，没有「不达标就退回重写」的强制循环

5 个独立 Agent 盲审解决这三个问题：
1. **盲审 = 零上下文污染**：每个 Agent 只拿到稿子正文，不知道关键词、竞品、大纲、客户。模拟的是「一个真实的搜索引擎用户 / AI 爬虫第一次读到这篇」的体验
2. **分维专审 = 深度**：5 个 Agent 各管一个维度，只审这一件事，审得更深
3. **95 分闸门 = 强制质量**：平均分不到 95 就打回，不存在「差不多就行」

---

## 五个盲审 Agent

每个 Agent 收到**完全相同的输入**：只有文章正文全文（含标题、H1、所有段落、表格、列表）。

每个 Agent 看到**不同的评分指令**：只评审自己负责的维度。

Agent 之间**不共享任何信息**，不知道彼此的存在，不知道这是第几轮审稿。

### Agent 1：信息增益与竞争力审查（0-20 分）

审查维度：这篇文章相对 Google 第一页已有内容，提供了多少**实质增量**。

评分标准：
- **20 分**：提供 ≥3 个头部页面完全没有的独家信息（一手经验/原创数据/独特框架/未公开规格），且每个都有实质深度
- **15-19 分**：提供 ≥3 个增量，但深度参差（有的只是提到，没有展开）
- **10-14 分**：提供 1-2 个增量，其他部分与竞品同质
- **5-9 分**：几乎复述竞品内容，偶有一两处不同措辞但不是实质增量
- **0-4 分**：纯粹是竞品内容的拼接重组，没有任何新信息

关键判定问题：
- 这篇文章有什么是 Google 第一页完全没有的？能列出具体的东西吗？
- 如果我是搜索这个主题的用户，这篇文章告诉我什么是我在别处看不到的？
- 文中是否有独家数据/案例/框架，还是全部来自公开可查的二手信息？

### Agent 2：E-E-A-T 与可信度审查（0-20 分）

审查维度：这篇文章读起来像不像一个**有经验的专业人士**写的。

评分标准：
- **20 分**：Experience（具体的一手经验描述）+ Expertise（准确的专业术语和判断框架）+ Authoritativeness（每个专业论断有可点击来源）+ Trust（数据有年份、来源可验证、利益透明）四项全部扎实
- **15-19 分**：三项扎实，一项偏弱（通常是 Experience 不足——缺一手素材）
- **10-14 分**：两项扎实，两项弱（常见的是 Expertise 有但 Experience 无，或来源模糊）
- **5-9 分**：只有 Expertise（专业术语用对了），但无一手经验、无具体来源、无信任信号
- **0-4 分**：读起来像 AI 生成的百科综述——无经验、无来源、无专业判断

关键判定问题：
- 读完后，我知道「谁写的、凭什么懂」吗？
- 文中的具体数据/论断有来源吗？点击能到达真正的原始出处吗？
- 有没有「我们实测发现」「客户案例里」这类第一手经验段落？
- 有没有「专家认为」「研究表明」这种模糊归因？（有就扣分）
- 利益相关时是否透明（如推荐自家产品时是否说明了）？

### Agent 3：可读性、结构与 GEO 就绪度审查（0-20 分）

审查维度：这篇文章是否**易于人类扫读 + 易于 AI 摘引**。

评分标准：
- **20 分**：answer-first 结构完美（开头直接回答，每个 H2 第一句给出该节核心答案）、段落自包含可独立摘引、表格/列表承载关键数据、H 标签层级清晰无跳跃、段落短（2-4 句为主）、无填充分段
- **15-19 分**：answer-first 基本做到，但有 1-2 个 H2 小节第一句在铺垫而非直接回答；部分关键信息嵌在大段文字里不易提取
- **10-14 分**：结构清晰但非 answer-first（先铺垫再给答案）；部分段落冗长（5+ 句），表格/列表使用不足
- **5-9 分**：段落长、结构松、关键信息被埋在文字里、H 标签层级有跳跃、没有表格/列表
- **0-4 分**：大量长段落、无清晰结构、无直接答案、读起来费力

GEO 特定检查：
- 开头 100 字内有没有直接回答主关键词背后的核心问题？
- 每个 H2 小节的第一句能否脱离上下文被单独引用、并且意思完整？
- 关键术语首次出现时有没有清晰定义？
- 对比/规格/参数数据在表格里还是在散文里？（表格 > 散文）
- 有没有「如上所述」「前面提到」这类跨段依赖？（有就扣分，因为破坏了段落自包含性）

### Agent 4：技术准确性与来源质量审查（0-20 分）

审查维度：这篇文章的**事实是否正确、来源是否可靠**。

评分标准：
- **20 分**：所有具体数字/规格/专业论断都有可点击的权威来源（A 级：标准/官方/学术占多数，B 级：行业报告辅之），来源年份在近 2-3 年内，数字与原文一致
- **15-19 分**：大部分有来源，但有 1-2 处来源等级偏低（C 级媒体）或年份偏旧但仍有参考价值
- **10-14 分**：来源覆盖不全，有 3+ 处数据/论断无出处；或来源年份明显过时（5 年+）仍当现状使用
- **5-9 分**：多数数据无来源，少数有来源但无法点击或来自不可靠的聚合站
- **0-4 分**：几乎没有可验证来源，或引用了明显的内容农场/AI 生成页

关键判定问题：
- 文中每个「X 是 Y」「数据显示 Z%」后面跟了可点击的来源链接吗？
- 来源是原始出处还是二手转述？（二手转述扣分）
- 来源发布年份标注了吗？够新吗？
- 有没有看上去合理但无法验证的「幽灵数据」？（比如一个很具体的百分比但没有来源）
- 有没有引用 Wikipedia 当终极来源？（应该顺着 Wikipedia 的 references 找到 A 级原始出处）

### Agent 5：语言质量与去 AI 痕迹审查（0-20 分）

审查维度：这篇文章读起来像不像**人写的、专业编辑过的**。

评分标准：
- **20 分**：完全读不出 AI 痕迹——无 AI 标志词汇（testament/landscape/delve/赋能/助力/打造）、无模糊归因、无套话开场、无伪金句、无填充短语、长短句自然混合、专业术语准确且一致
- **15-19 分**：几乎无 AI 痕迹，偶有 1-2 处可改进（如一个段落略公文腔、一处措辞可更自然）
- **10-14 分**：整体人感不错，但有 3-5 处 AI 痕迹（如一两处「值得注意的是」「综上所述」、一处模糊归因、一两处「不仅是X更是Y」结构）
- **5-9 分**：AI 痕迹明显——多处 AI 标志词汇、套话开场、多处模糊归因、混入了典型的 AI 措辞模式
- **0-4 分**：一眼 AI——从头到尾都是模板化措辞、套话堆砌、每段读起来都像 ChatGPT 默认输出

B2B 技术内容特殊检查：
- **保留**：规格/参数对比表、技术粗体标注、步骤编号列表 → 这些是信息结构，不是 AI 痕迹
- **去除**：「值得注意的是」「综上所述」「在这个数字时代」「从本质上讲」→ 零信息量短语
- **去除**：「不仅是 X，更是 Y」「它不仅能 A，还能 B，更能 C」→ 为对称硬凑
- **去除**：「赋能」「助力」「打造」「不断」「深入」→ 中文 AI 高频虚词
- **去除**：「希望对您有帮助」「让我们深入了解」→ 聊天套话
- **判据**：删了这句话，信息有没有减少？有减少 → 保留；没减少 → 删除
- 每段问自己：一个真实的行业专家会这样写吗？还是会写得更直白、更具体？

## 评分汇总与闸门判定

### 汇总规则
1. 收集 5 个 Agent 的评分
2. 计算平均分：`平均分 = (Agent1 + Agent2 + Agent3 + Agent4 + Agent5) / 5`
3. 检查是否有单个维度低于 15 分（即使平均分 ≥ 95，单维 < 15 也视为不通过）

### 闸门判定
```
平均分 ≥ 95 且 所有单维 ≥ 15 → ✅ 通过，进入 Step 7 交付
平均分 ≥ 95 但有单维 < 15 → ❌ 打回，重点修复该维度
平均分 < 95 → ❌ 打回，根据所有 Agent 的扣分点逐条修复
```

### 打回修订流程
1. 汇总所有 Agent 的**具体扣分点**（不是「信息增益不够」，而是「缺少 X 应用场景的真实案例，竞品 A 和 B 都有但你只是泛泛提了一句」）
2. 按扣分严重程度排序，从扣分最多的维度开始改
3. 改完后，重新跑一轮完整的 5 Agent 盲审（Agent 是全新实例，不知道上一轮的评分）
4. 最多迭代 5 轮。5 轮后仍未达标 → 如实报告「经 5 轮迭代仍未达到 95 分，以下是未解决的维度：[列出来]。建议：[补一手素材 / 换角度 / 降低难度预期]」

## 执行机制（给主 Agent 的指令）

你是主 Agent（vibio-content）。以下是执行多 Agent 盲审的方式：

### 发起盲审
使用 `Agent` 工具同时启动 5 个独立 Agent，每个设置为 `run_in_background: false`（等待全部返回）。

每个 Agent 的 prompt 模板如下（只替换 `{文章正文}` 和 `{审查维度}`）：

```
你是 SEO 内容质量审查员。你将收到一篇文章的完整正文。
你的唯一任务：按给定的评分标准，只审查 {审查维度} 这一个维度，给出 0-20 的分数和具体的扣分点。

重要约束：
1. 你只能看到这篇文章的正文，没有其他任何上下文
2. 你不知道这篇文章的目标关键词、竞品分析、内容大纲或客户信息
3. 你的评分必须基于文章本身的内容质量，而不是与其他文章比较
4. 每个扣分点必须具体：指出文中的哪句话/哪个段落有问题，为什么扣分

{该维度的详细评分标准}

文章正文：
---
{文章正文}
---

请按以下格式输出（严格 JSON）：
{
  "dimension": "{审查维度}",
  "score": <0-20的整数>,
  "passed": "<单维≥15为true>",
  "strengths": ["具体做得好的地方"],
  "weaknesses": ["具体扣分点，每条指出文中段落/句子和扣分原因"],
  "must_fix": ["必须修改的关键问题，不改无法通过"]
}
```

### 汇总与判定
1. 等 5 个 Agent 全部返回
2. 计算平均分
3. 检查单维最低分
4. 按闸门判定规则决定通过或打回

### 修订
1. 将 5 个 Agent 的 `weaknesses` 和 `must_fix` 合并去重
2. 逐条修改文章
3. 修改完成后，重新发起 5 Agent 盲审（全新实例）

### 迭代上限
最多 5 轮。每轮记录：
- 轮次编号
- 5 个 Agent 的分数
- 平均分
- 是否通过
- 如果不通过，本轮集中修复了哪几个问题

5 轮后仍未通过 → 停止迭代，如实报告。
