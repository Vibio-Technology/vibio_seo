# Vibio SEO — LINK 引擎

内链帮助用户和抓取系统发现、理解并访问相关页面；外链应来自真实资格、关系或值得引用的资产。品牌提及可作为 PR、品牌需求和潜在线索证据，但没有可靠公式把它换算成链接、排名或 AI 引用倍数。

## 执行边界

- 新页面至少需要一个真实、可抓取的站内入口；donor 数量由信息架构和用户任务决定。
- 买链、PBN、批量客座农场、隐藏链接和操纵性交换属于禁区；遵循 Google link spam 政策。
- 对外注册、发信、发布和代表企业回应属于外部动作。默认由 agent 做研究、事实核验、清单和草稿，写入 `pending-human-review`；只有用户明确授权且工具/账号允许时才执行。
- 先处理当前主导约束。内链通常可控，但若用户明确要外链且站内无阻断，不强制先做一轮无关内链工作。

## 执行流程

### 0. 续接项目状态

读取 `.vibio/project.md`、content/keyword/link/outreach trackers 和 changelog。复用已有页面库存、业务目标、上次审计与历史联系人，避免重复研究。使用当前环境的文件能力，不绑定特定工具名。

### 1. 诊断链接问题

按 `references/link-architecture.md` 检查：

- sitemap、crawl、导航和内容库存之间的孤儿/发现差异；
- 优先商业页是否有相关、可见、可抓取的入口；
- 断链、重定向链、错误目标和模板性过度链接；
- 锚文本是否准确预告目标，是否存在误导或操纵；
- 新内容是否完成从旧页链入的发布回填；
- 多页争抢同一任务时是否应该先合并或重构。

入链数、点击深度和链接密度只在同站同类页、真实用户路径中解释，不使用 `<5`、`>3` 或每若干字一个链接的通用失败线。

有代码/模板权限时可直接修并验证；只有 URL/CMS 权限时给精确 donor、位置、锚、目标和验证清单。

### 2. 判断外链准备度

页面 outreach 前确认：

- 目标页可索引、内容稳定并能兑现标题/邮件承诺；
- 有清楚的编辑理由，例如原创数据、工具、规格参考、案例、图像、专家证据或独特资源；
- 目标网站与其受众相关，链接会帮助读者；
- 目标、成本、人工负责人和结果记录方式明确。

不使用“至少 5 篇内容”或某个 authority phase 作为硬门槛。真实客户、伙伴、协会、认证、展会或媒体关系在任何阶段都可做，只要企业确有资格、页面有价值且做法合规。没有可引用理由时，先用 `vibio-content` 或产品/数据工作补资产。

### 3. 建立机会池

按与业务和编辑价值的贴近程度排序：

1. 真实客户、供应商、伙伴、协会、认证、展会、商会和目标市场高质量目录；
2. 已有但未链接的真实品牌/产品/研究提及；
3. 资源页、贸易出版物、专家贡献和行业媒体；
4. 原创研究、工具、数据、案例和数字 PR；
5. 竞品 link gap。

竞品 gap 只是候选发现：逐站验证它为何链接竞品、是否仍活跃、受众是否相关、你的资产是否提供同等或更高价值。不能因为一个域链接多个竞品就称为“可复制”。

### 4. 研究与 outreach 备料

每个 prospect 记录：来源、相关受众、具体页面、编辑理由、联系人/渠道、事实核验、拟推广资产、个性化角度、风险和状态。邮件必须准确说明为什么该资源对对方读者有用，不能伪造关系、引用、数字或紧迫性。

转化预期用团队自身历史 cohort；没有数据就把回复率/获链率设为待测，不套通用百分比。agent 默认只推进到 `pending-human-review`。

### 5. 安全与质量

- 检查付费、互惠、赞助、UGC 等链接属性是否符合 Google 政策；
- 不操纵外链锚文本或按固定比例“稀释”；
- 不以 DR/DA 单分数决定质量，结合相关性、编辑控制、真实受众、引荐和垃圾模式；
- Disavow 只在有大量人为制造的垃圾链接，且已导致或很可能导致 manual action 时考虑；不例行提交；
- 记录失败、拒绝和无回复，防止选择性汇报。

### 6. 验证与写回

内链修复验证渲染 HTML、目标状态和 crawl/导航入口；外链验证实际 URL、可访问性、rel 属性、上下文和目标。更新 trackers 与 changelog，保留历史。

效果观察窗口由抓取、查询数据量、页面成熟度和业务周期决定。使用 `vibio-review` 比较处理资产与可比页面的发现、目标查询、引荐、合格访问和业务结果，不承诺固定月份。

## 专家能力与降级

- `references/link-architecture.md`：内链发现、拓扑、锚与回填；
- `references/backlink-playbook.md`：资产、机会、outreach、安全和测量；
- `seo-backlinks`：引用域、锚、竞品 gap 与风险数据；
- `seo-dataforseo`：第三方口径的外链/提及候选数据；
- `seo-firecrawl`：有界 crawl、孤儿与断链；
- `seo-google`：可用的 GSC 页面/链接/表现数据。

所有外部能力均可选。缺失时用 sitemap、渲染 HTML、源码、公开页面和用户导出数据降级；没有竞品外链数据就明确缺失，不编域名或指标。

## 下一步路由

| 条件 | 下一步 |
|---|---|
| 缺少能完成任务的页面/资产 | `vibio-keyword` 验证需求，`vibio-content` 生产资产 |
| 内链需要模板/组件修改 | `vibio-fix` 按栈落地并验证 |
| 外链/内链已执行，需评估增量 | `vibio-review` 按项目数据成熟度复盘 |
| 需要竞品持续监控 | 使用 `references/competitive-war-room.md`，数据能力缺失时降级为定期人工样本 |

## 不要做

- 不发布没有任何可抓取入口的孤儿页。
- 不买链、碰 PBN/链接包、批量客座农场或其他 link spam。
- 不在没有真实编辑理由时批量 outreach，也不把内容数量/抽象 phase 当唯一准备度标准。
- 不把竞品链接域直接当可复制名单。
- 不发虚假或无关模板邮件，不代表企业进行未授权外部动作。
- 不操纵外链锚文本，也不强制站内锚轮换比例。
- 不例行 disavow，不因未经证实的负面 SEO 就提交。
- 不把品牌提及、DR/DA、链接数量换算成固定排名或 AI 分数。
- 不承诺若干链接换若干名次或在固定时间见效。
- 不覆盖历史 tracker 或重复联系已拒绝/已处理的对象。
