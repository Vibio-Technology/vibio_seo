# 内链架构手册 — 站内链接的统一操作规范

内链是站内唯一完全可控、零成本、见效相对快的权重杠杆，也是 AI 搜索判断主题权威的核心信号之一——但它常年被当成"发文章时顺手加两个链接"。本手册把内链升级为**一个持续运营的系统**：拓扑怎么搭、锚文本怎么写、新页发布时怎么插链、怎么审计维护。

分工声明：本文件是内链的**操作层权威**。`authority-cascade.md` §3（级联阶段的内链时序）和 `semantic-networks.md` §4（实体网络的三层链接拓扑）是本文件在两种策略场景下的特化，规则冲突时以本文件为准。

---

## 一、内链在做什么（四个作用，缺一个都亏）

1. **收录与爬取** — 爬虫沿链接发现页面。孤儿页（0 条站内入链）即使在 sitemap 里也几乎不会有好排名；点击深度 4+ 的页面爬取频率显著衰减（业界测算约 -70%）。
2. **权重导流** — 链接传递权威。高流量页是"供体"（donor），弱势钱页是"受体"（acceptor）；不设计导流，权重就淤积在博客层，钱页拿不到。
3. **主题信号** — 簇内互链告诉 Google"这些页面共同构成一个主题的完整覆盖"。锚文本是最直接的相关性说明。
4. **AI 搜索上下文** — AI 引擎按语义关系选引用源：成簇互链的内容比孤立页面获得的 AI 引用显著更多（业界研究约 3.2 倍），因为链接结构强化了实体与主题的关联判断。

## 二、站点拓扑规范

### 三层结构（默认模型）

```
Tier 1  钱页/支柱页（服务页、产品类目页、核心指南 hub）
   ↕ 双向
Tier 2  簇页（应用场景、对比、选型指南）——链回支柱 + 横向互链
   ↕ 双向
Tier 3  长尾页（规格、FAQ、单点问题）——链上簇页，喂权重
```

**硬规则：**
- **钱页距首页 ≤3 次点击。** 每次审计先量点击深度，超深的钱页立刻从高权重页补链拉近。
- **支柱 ↔ 簇双向链接**，不是单向：支柱页链向每个簇成员，每个簇成员链回支柱。
- **允许跨簇链接**（真实相关时）——严格封闭的 silo 已过时，"联网的 hub"是现行最优。
- **正文上下文链 > 导航/页脚/侧栏链。** 模板化链接（全站页脚）传递的相关性信号远弱于正文中带上下文的链接。导航保证可达性，正文链负责权重与相关性——两套系统，别混算。

### Donor–Acceptor 权重导流（每季度跑一遍的动作）

1. 从 GSC 导出曝光/点击最高的页面（donors，通常是博客/指南）。
2. 列出需要权重的弱势钱页（acceptors：排名 11-20 的商业页、新服务页）。
3. 每个 donor 向语义相关的 acceptor 添加 2-3 个正文上下文链。
4. 记录到 changelog，2-6 周后 REVIEW 复测 acceptor 排名。

## 三、锚文本规则

- **站内精确匹配锚安全且有益**（与外链正相反，站内不会因精确锚被罚）。目标页的核心词就该出现在指向它的锚文本里。
- **每个重要页面维护 4-6 个锚变体**（"锚库"）：精确匹配 / 部分匹配 / 描述性长句 / 品牌+词。不同来源页轮换使用——指向同一页的锚全站一字不差，信号会退化成模板噪音。
- **锚文本必须预告目标页内容**：读者点之前就知道会看到什么。禁用"点击这里""了解更多""this article"。
- **锚要嵌进自然句子**，为插锚硬造的句子在盲审（Agent 5）会被当 AI 痕迹扣分。
- 页脚/导航/面包屑的锚不计入锚多样性——它们是导航系统，不是编辑信号。

## 四、数量与密度

| 规则 | 数值 | 说明 |
|---|---|---|
| 正文上下文内链 | 每 1000 字 2-5 个 | 2000 字文章即 4-10 个，取值看内容自然程度，不必卡满上限 |
| 指向同一目标页 | 每 500-800 字最多 1 个 | 沿用 `authority-cascade.md` §3 |
| 单页总链接（含导航） | < 150 | 超出即稀释 + 可读性差 |
| 新页面入链下限 | ≥ 2-3 条（发布当天） | 见第五节发布协议 |
| 钱页入链目标 | 站内前 10% | 商业价值越高，入链越多 |

过密比过疏更常见也更伤：链接堆到读者点不过来 = 每条的价值都被稀释。判据永远是"读者此刻需要这个跳转吗"。

## 五、新页面发布协议（强制仪式，不是可选项）

**每次发布新页面，当天完成以下四步。跳过这一步，新页就是孤儿页，前面写作的所有功夫打对折。**

执行方式随编辑模式（与 `stack-detection.md` 四分法一致）：**code/template** 直接改源文件并验证；**CMS-paste** 由 agent 产出可粘贴片段 + 精确定位（哪篇文章、正文第几段、锚文本原句），人工登录后台粘贴；**URL-only** 同样给精确清单。

1. **反向插链（最重要、最常被跳过）**：找 2-3 个相关的**已有页面**，在其正文中添加指向新页的上下文链接。找 donor 的方法（按优先序）：
   - `.vibio/trackers/content.md` 里同簇的已发布页
   - GSC 曝光最高且主题相关的页（权重最大的 donor）
   - `site:域名 关键词` 搜出的相关旧文
   - 代码库/CMS 里 grep 新页主题词，找到提及处直接加链
2. **新页链出**：正文中链向所属支柱页 + 1 个相关钱页 + 1-2 个同级簇页，锚文本按第三节规则。
3. **更新支柱页**：新页所属的 hub/支柱页添加指向新页的链接（保持"支柱链向全部簇成员"不变式）。
4. **登记**：把插链动作记入 `.vibio/changelog.md`（Type: FIX change）；内容简报流程中（`seo-content-brief` / WRITE 阶段 4）就应预先写明"这篇发布时从哪 2-3 页插链进来、链出到哪几页"，把本协议前置到规划期。

**级联刷新**（沿用 `authority-cascade.md`）：当某页获得排名后，它的内链权重变大——回头检查它是否已链向当前阶段的目标页；新一阶段页面发布后，回头更新上一阶段已排名页面的链出。

## 六、审计与维护

### 检测项与方法

| 检测项 | 判定 | 方法 |
|---|---|---|
| 孤儿页 | 站内入链 = 0 | 全站爬取（`seo-firecrawl`）结果 vs sitemap vs GSC 有曝光页三方对比：在 sitemap/GSC 里但爬不到的就是孤儿 |
| 弱链接页 | 入链 < 5 | GSC Links 报表 → Top Internal Links 升序；或爬取数据按入链数排序 |
| 超深钱页 | 点击深度 > 3 | 爬虫的 crawl depth 字段 |
| 过链接页 | 单页链接 > 150 | 爬取数据；修法是删弱相关链、把次要链接移入"相关阅读"块 |
| 断内链 / 重定向链 | 4xx / 301 链条 | 爬取报告；断链直接修，内链一律改指最终 URL |
| 锚文本蚕食 | 同一锚指向多个不同页 | 导出全站锚文本分组检查；同锚多目标会让 Google 分不清哪页是该词的主答案 |

**无爬虫工具时**（代码栈）：静态站/Next.js 直接 grep 源码统计 `<a href` / `<Link` 的入链分布——注意 grep 对动态路由/CMS 驱动的导航会漏检，程序化生成页面的孤儿判定要结合渲染后 HTML（curl 抓部署后页面）或如实告知结果不完整；小站手工核对 sitemap 与导航可达性。URL-only 模式给出人工核对清单。

### 修复优先级

孤儿钱页 > 断内链 > 超深钱页 > 弱链接的优先页 > 锚蚕食 > 过链接页 > 其他。

## 七、内链与 AI 搜索

- AI 引擎按**语义关系网**选引用源，不数链接条数。簇结构完整（支柱全覆盖 + 簇页互链）的站点，其中任一页面的段落被判定"可引用"的概率都更高。
- 段落级引用集中在页面前 30%（多项研究：约 44-55% 的 AI 引用来自内容前三分之一）——内链的"相关阅读"块别堆在文首，正文前三分之一优先留给可摘引的直接答案（与 `geo-content-patterns.md` 一致）。
- 锚文本为 AI 提供实体关系标注：「carbon fiber for drone frames」这样的锚比「点击这里」多传递一整层实体信息。

## 八、内链健康快照（写回 `.vibio/trackers/links.md`）

首次内链审计时创建（格式见 `state-templates.md`）：孤儿页清单、Top 钱页入链数、待回填队列、上次审计日期。发布协议欠的账（该插未插的链）也记在待回填队列里，周度清零。

## 九、节奏

| 时机 | 动作 |
|---|---|
| 每次发布 | 第五节发布协议四步（当天） |
| 每周（Block 3, 30-60m 内） | 清空待回填队列；确认新页已被收录 |
| 每月 | GSC Top Internal Links 扫弱链接页；孤儿页快查 |
| 每季度 | 全量审计（第六节全部检测项）+ Donor-Acceptor 导流 pass + 更新 links tracker |

## 不要做

- 不发布没有入链的新页面——发布协议四步是仪式，不是建议。
- 不用"点击这里/了解更多"做锚文本——锚必须预告目标页内容。
- 不给同一目标页全站复制同一句锚——维护锚库轮换。
- 不把导航/页脚链接当成内链建设——正文上下文链才传递相关性。
- 不为插链硬造句子——锚要自然嵌入，否则盲审当 AI 痕迹扣分。
- 不搞严格封闭 silo 禁止跨簇链接——真实相关就链。
- 不堆链接——每 1000 字 2-5 个，单页 <150，判据是读者需要。
- 不修完内链不记 changelog——REVIEW 模式要凭记录判断导流是否见效。
