# Authority Cascade — 证据驱动的机会级联

机会级联的目的，是让早期投入产生可验证信号，再用这些信号决定下一批页面和权威投入。它不是“先做固定 KD 区间，达到固定排名比例才能升级”的排名公式。

Google 没有公开“主题权威分”或 KD 阈值。KD/DR/DA 是第三方指标，只能在同一数据供应商、同一口径下辅助比较，不能当 Google 门槛或结果承诺。

## 一、四层级联

```text
Layer 0：资格与测量
  -> Layer 1：最接近收入、证据最强的机会
  -> Layer 2：从真实查询和买家任务扩展主题覆盖
  -> Layer 3：对已验证的竞争机会追加内容、产品证据、PR 与链接资源
```

层级描述投入依赖，不等于关键词难度等级。同一主题中可并行存在多个层级；不需要等待所有 Layer 1 词“进前十”才开始下一层。

### Layer 0：资格与测量

先确保：

- 重要页面可抓取、可索引、返回正确状态并有一致 canonical；
- GSC 已验证，GA4/CRM 或其他业务测量能按自然落地页观察结果；
- 目标国家、语言、买家与主转化已定义；
- 已有页面、关键词和内容库存可追踪。

发现索引阻断时优先修复。单纯缺少 sitemap、Schema 或某项评分，不自动构成阻断，按真实影响判断。

### Layer 1：高贴合、低遗憾机会

优先选择同时满足下列条件的查询族/页面：

- 来自 RFQ、CRM、站内搜索、GSC 或付费 search terms 的真实买家语言；
- 商业路径短，企业确实能满足页面承诺；
- 目标市场 SERP 的人群与页型匹配；
- 站点有独有规格、案例、数据、工具或一手经验可提供；
- 实施成本可控，失败后仍会改善核心商业页面或信息架构。

典型产物是产品/服务、应用、规格、选型、对比和证据页。是否“长尾”只是现象，不是入选条件。

### Layer 2：由数据扩展覆盖

Layer 1 上线并获得足够抓取/曝光后，从真实数据扩展：

- GSC 暴露出的相关查询和意图缺口；
- 销售/RFQ 的高频问题与异议；
- 目标地区 SERP 中尚未完成的买家任务；
- 能为商业页提供必要解释、证明或导航的支持内容；
- 已出现 cannibalization 的词族合并与页面重映射。

不为同义词或每个 fan-out 变体批量建页。只有用户任务、页型或产品能力确有差异，才拆成独立 URL。

### Layer 3：竞争投资

只有当需求和业务路径已被证明，才追加高成本资源：

- 原创研究、工具、数据集、深度案例或可引用资产；
- 模板/产品数据系统和程序化页面治理；
- 数字 PR、伙伴/协会/媒体关系和相关编辑型链接；
- 国际化、视频、交互或大规模内容更新。

“需求已证明”可以来自相关自然曝光、合格广告 search terms、销售数据或同类页面 cohort，不要求达到固定名次。链接来自真实关系或资产价值时，任何阶段都可以获得；不得因为层级较早而拒绝自然的客户、伙伴、协会或媒体链接。

## 二、排序方法

每个机会保留以下维度及证据，不强行相加为万能分数：

| 维度 | 要回答的问题 |
|---|---|
| 商业价值 | 对合格线索、订单、留存或销售效率有何作用？ |
| 需求证据 | GSC、RFQ、CRM、Ads search terms、SERP 或工具数据证明了什么？ |
| 意图/页型匹配 | 目标地区结果是否支持计划中的页面类型？ |
| 信息增益 | 企业能提供什么前排页面没有的一手内容或体验？ |
| 当前立足点 | 是否已有相关曝光、页面、内链、品牌需求或引用？ |
| 成本与依赖 | 需要产品、开发、翻译、法务、设计或外联多少投入？ |
| 可测性 | 上线后能否判断继续、停止或加码？ |

输出 `now / next / later / reject`，并写一句决策理由。第三方 KD 可作为“竞争成本”证据之一，但必须标明工具、国家、日期；不得跨工具把 KD 分档当普遍事实。

## 三、级联内链

内链服务发现、理解和用户导航：

```text
商业/支柱页
  <-> 规格、应用、选型、对比与案例页
  <-> 相关产品、证据和下一步动作
```

- 新页面发布时，从最相关且可抓取的已有页面建立至少一个真实入口；数量由信息架构决定。
- 支持页在语境合适时链向商业页，商业页也应链回深入解释和证据。
- 锚文本准确描述目标，不要求精确匹配关键词，也不设“每若干字一个链接”的密度规则。
- 新节点上线后清理回填队列；不要只给新内容链出、不从旧内容链入。

完整规则见 `references/link-architecture.md`。

## 四、升级与停止闸门

每个机会在启动前写明继续条件和停止条件。根据数据量选择窗口，不套固定月份。

### 可以加码

- 页面已按规格上线且被抓取/收录；
- 在目标市场出现相关曝光或合格访问；
- 查询、线索或用户行为支持原意图；
- 同类页面 cohort 显示该内容/页型有增量；
- 更高成本投入有明确的信息增益与测量方案。

### 先诊断

- 没抓取或没收录：查技术资格与发现路径；
- 有曝光但查询错配：重做意图、标题或页面定位；
- 有目标访问但无业务结果：查承诺、信任、表单、产品适配和测量；
- 同一词族多页轮换：查 cannibalization 与页面归属；
- 前排结果/需求发生变化：重新采样 SERP，不继续按旧 KD 投资。

### 停止或降级

- 目标市场 SERP 和真实买家语料持续否定需求；
- 企业无法提供差异化内容或兑现承诺；
- 达到预先约定的成本/观察窗口后，领先与业务信号均不支持继续；
- 更重要的技术、产品或测量瓶颈尚未解决。

## 五、示例：B2B 材料站

不要预填虚构的搜索量和 KD。按证据展开：

1. RFQ 反复出现某材料规格与认证组合，目标市场 SERP 以供应商/规格页为主：建或强化对应规格商业页，归 Layer 1。
2. 上线后 GSC 出现选型、对比和应用查询：将能共同完成任务的问题合入主页面，真正需要独立任务的建支持页，归 Layer 2。
3. 销售确认该主题带来合格机会，但前排结果有实测数据和行业引用：投资测试报告、选型工具、案例与相关 PR，归 Layer 3。
4. 泛词量大但人群混杂、商业路径弱：保留 `later`，不因“头部词”身份自动投入。

## 六、与各模式的连接

| 模式 | 使用方式 |
|---|---|
| `vibio-plan` | 用层级和依赖排路线图，不把所有动作塞进第一周 |
| `vibio-keyword` | 为词族标 `now/next/later/reject` 和证据，不再写固定 `cascade_phase` 数字 |
| `vibio-content` | 优先生产有真实需求、商业路径和信息增益的页面 |
| `vibio-link` | 把高成本外联投向已验证资产，同时接受任何阶段的真实编辑型链接 |
| `vibio-review` | 用预先定义的领先/业务信号决定加码、调整或停止 |
| `references/predictive-seo.md` | 用站内 cohort 和情景区间估计投入，不套 KD/CTR 通用常数 |

## 七、不要做

- 不承诺“低 KD 一定更快”，也不把某工具的 KD 当 Google 规则。
- 不制造搜索量、排名周期、升级比例或“主题权威分”。
- 不等早期所有页面排名后才处理已验证的高价值机会。
- 不为证明主题覆盖而批量生产相似页面。
- 不以“阶段太早”为由拒绝真实、相关、编辑型链接。
- 不用精确匹配锚文本或固定内链密度操纵信号。
- 不把一个市场/主题的学习原样复制到另一个市场；每次重新验证需求和 SERP。
