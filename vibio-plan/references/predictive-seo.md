# SEO 预测与投资决策

本文件用于回答“这个 SEO 机会值得投入吗”，不是预测某个词必然排第几。搜索结果、竞争、抓取和需求都会变化；可靠的产出应是**证据链、情景区间、关键假设和下一次校准点**，而不是伪精确的排名概率。

## 一、预测原则

1. **先用站点自己的历史。** GSC、GA4、CRM、已发布页面和实际成本，比通用 CTR、转化率或“域名权威分”更能预测该站结果。
2. **缺失不是零。** 拿不到搜索量、转化率或引用域时标 `unknown`，降低置信度；不能按 0 填充后继续算。
3. **第三方指标只在同一工具内比较。** KD、DR、DA、流量估算的口径由供应商定义，不能当 Google 指标，也不能跨工具直接换算。
4. **区间优于点估计。** 至少给保守、基准、进取三种情景，并展示哪个输入最影响结果。
5. **预测必须可回测。** 每个预测写明建立日期、数据窗口、假设、观察窗口和复盘日期；到期用真实结果校准。
6. **不把相关性写成因果。** Schema、字数、域龄、内容数量和外链数量都不能单独推出排名结果。

## 二、最低输入集

没有以下数据时，可以做机会诊断，但不能给高置信度流量或收入预测：

| 输入 | 首选来源 | 缺失时的处理 |
|---|---|---|
| 目标市场、语言、业务目标 | 业务资料 / 用户确认 | 标注假设，不能用全球口径代替 |
| 查询或主题需求 | GSC、Keyword Planner、可核验第三方数据 | 只做定性需求判断，不编搜索量 |
| 目标地区真实 SERP | 带日期、国家、语言、设备的实时采样 | 标 `SERP-unverified`，不判可竞争性 |
| 站点现有搜索表现 | GSC 页面/查询/国家/设备数据 | 用可比页面历史；没有则低置信度 |
| 页面与竞争证据 | 渲染 HTML、页面内容、引用域导出 | 明确未检查项 |
| 转化与成交 | GA4 落地页、CRM 线索/订单 | 只算点击或给盈亏平衡条件 |
| 交付成本 | 实际工时、内容、开发、外联成本 | 给投入清单，不计算 ROI |

所有数值都记录来源、时间范围、地区、设备、品牌/非品牌过滤和是否为估算。执行细则见 `references/evidence-policy.md`。

## 三、机会判断

逐项保留证据，不把所有信号压成一个“SEO 分数”。

### 1. 商业贴合

- 查询对应的搜索者是否是目标客户？
- 页面能否承接到产品、询价、试用、门店或其他真实转化？
- 企业是否真的能提供页面承诺的规格、地区、服务或证据？

商业路径不成立时，搜索量再高也应降级。

### 2. 需求证据

按强度记录：自有 GSC 查询、站内搜索/RFQ/CRM、广告 search terms、目标市场 SERP/Autocomplete、Keyword Planner 或第三方量。广告数据只能证明付费环境中的查询和转化线索，不能直接证明自然排名难度或自然 CTR；用法见 `references/paid-search-intelligence.md`。

### 3. 意图与页型

抽样目标地区 SERP，记录主要页型、结果承诺、SERP 功能和变化。计划中的页面类型若与结果页主流意图不一致，先调整页面或放弃，不靠加字数解决。

### 4. 当前立足点

优先看该站自己的相邻主题：

- 已有曝光或排名的页面/查询；
- 同类页面从发布、首次抓取、收录到稳定曝光的历史；
- 内链入口、内容证据和真实引用域；
- 页面是否已经满足抓取、索引和体验基本条件。

`site:` 结果数不是可靠的收录总量，域龄也不是排名预测器；不得用它们替代 GSC 或真实 SERP 证据。

### 5. 竞争与信息增益

比较前排页面是否满足用户任务、是否有一手证据、产品数据、实测、工具、案例或更合适的交互。所谓“弱结果”必须基于可观察缺口，不能因为对方没有 Schema、没有 Wikipedia、字数少或品牌不熟悉就判定可击败。

输出每个机会的 `invest / test / hold / reject`：

- `invest`：商业路径清晰，需求与意图已验证，站点有现实立足点，并能交付显著信息增益。
- `test`：价值可能高，但一到两个关键假设尚未验证；先做低成本页面或广告/销售语料实验。
- `hold`：依赖项未完成，例如索引阻断、缺少承接页、测量缺失。
- `reject`：人群、意图、市场或产品能力明确错配。

## 四、流量情景模型

可使用以下恒等式，但每个变量必须来自本项目数据或明确的区间：

```text
自然点击 = 可测查询需求 × 自然曝光份额 × 实际 CTR
```

### 已有页面

首选 GSC 建立同页面、同国家、同设备、同品牌过滤下的基线。用历史区间或相似页面 cohort 估计曝光和 CTR，不套公开“第 1 名 CTR”表。平均排名是聚合指标，不能把单一平均位置直接代入通用曲线。

### 新页面或新主题

找站内已发布的可比 cohort：相近页型、意图、市场、内部链接和品牌成熟度。用这些页面的分位区间构建三种情景。没有可比 cohort 时只给：

- 能观测到的需求上界；
- 达到盈亏平衡所需的最低点击/线索；
- 哪些早期信号会支持或否定继续投入。

AI Overviews、精选摘要、购物结果、地图和视频会改变点击分布，但不得套固定折扣。记录真实 SERP 形态，并用该站 GSC 的前后数据或受控实验估计影响。Google 对 AI 功能的官方建议仍是基础 SEO 和有帮助的内容，不需要独立“AI 排名公式”：https://developers.google.com/search/docs/fundamentals/ai-optimization-guide

## 五、业务价值与盈亏平衡

有第一方漏斗数据时可计算：

```text
预期价值 = 自然点击
         × 落地页转化率
         × 合格线索率
         × 成交率
         × 单次成交贡献价值
```

电商可替换为购买率与订单贡献；订阅业务可使用激活率、付费率和经批准的客户价值口径。所有比率必须匹配相同市场、页型和时间窗口。

没有完整数据时，反推盈亏平衡比制造行业默认值更可靠：

```text
盈亏平衡所需成交数 = SEO 总成本 / 单次成交贡献价值
盈亏平衡所需自然点击 = 盈亏平衡所需成交数 / 实测点击到成交率
```

若点击到成交率未知，就把它作为敏感性变量列成区间，不填“B2B 默认 2%”之类的通用数字。归因限制和增量判断见 `references/roi-attribution.md`。

## 六、时间与观察窗口

不承诺“低 KD 三个月上首页”。观察窗口由以下项目数据决定：

- 同类页面的抓取与收录延迟；
- GSC 数据量与查询波动；
- 发布/修复是否已在渲染结果中生效；
- 需求季节性、销售周期和重大站点变更；
- 是否有可比对照页或分批上线。

为每项投资设三个检查点：

1. **实施校验**：页面、状态码、canonical、内链、测量是否按规格生效。
2. **领先信号**：抓取、收录、目标国家曝光和相关查询是否出现。
3. **业务信号**：合格自然访问、线索质量、辅助转化或成交是否改善。

没有足够数据时判 `insufficient evidence`，不能把“未显著变化”写成“无效”，也不能说“再等等一定会涨”。

## 七、实验与校准

- 优先分批发布同类页面，保留未处理的可比组；重大站点变更使用中断时间序列或差分思路。
- 标题、内容、内链等实验一次只改变可解释的变量组，预先写主指标与护栏指标。
- 避免只挑成功页面复盘；记录所有进入 cohort 的页面。
- 每次复盘把预测区间和实际结果写回 `.vibio/`，更新该站自己的先验，而不是修改成另一套行业常数。

统计与实验设计见 `references/seo-experimentation.md`。

## 九、硬停止条件

- 目标市场或搜索意图未验证；
- 关键页面不可抓取、不可索引或无法承接业务；
- 所需数据不可得，却被要求给确定性排名、流量或收入承诺；
- 预测依赖虚构搜索量、通用 CTR、通用转化率或未经说明的第三方分数；
- 站点正在迁移、重大改版或测量中断，无法建立可比基线；
- 查询涉及新闻、突发趋势或高度波动结果，却没有实时数据。

此时交付“如何获得可预测数据”的最短实验，不交付漂亮但不可证伪的数字。
