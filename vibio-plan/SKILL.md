# Vibio SEO — PLAN 引擎

把一个站点、业务或代码库变成一份按依赖关系排序的 SEO 执行计划。你交付的是可执行计划，不是咨询 PPT。

核心约束：**先看真实项目，再下计划；先找一个主瓶颈，再排优先级；不承诺排名。**

---

## 执行流程

完整方法在 `references/operating-system.md`，输出结构在 `references/delivery-template.md`。按以下阶段跑：

0. **读项目记忆。** 开工前用 `Read` 直接读项目根的 `.vibio/`（格式约定见 `vibio-memory`）：**有记忆就走延续模式**——别从头 kickoff，先认清当前阶段、已完成项、上次主瓶颈，只输出下一段操作切片（见 operating-system.md 延续规则）；**没有**就是新项目，正常往下，收工时用 `Write` 建记忆。

1. **Kickoff** — 从真实产物（代码库、线上页面、sitemap、内容清单、文档）+ 业务上下文推导：业务模型、主要转化（购买/询盘/demo/WhatsApp/电话）、目标市场与语言、技术栈、域名新旧、是否已有 GSC/GA4、现有内容库、每周可投入工时。优先用产物，只问会实质改变排序的阻塞性问题。

2. **分类** — 选一个主类：`新站` / `存量弱站` / `存量内容站` / `电商目录站` / `服务/B2B 站`。再加一个次级标签：`技术债` / `内容债` / `权重缺口` / `度量缺口` / `国际化`。

3. **诊断主瓶颈** — 只选一个（技术基线 / 关键词架构 / 内容系统 / 优化刷新 / 权重 / 度量），围绕它排序一切。给出 **90 天目标**：具体、可完成、绑定到页面/系统，绝不是排名承诺。
   - 好：「建立干净的技术基线，定义关键词架构，发布前 12 个高意图页面。」
   - 坏：「主词排到第一。」

4. **排 90 天路线图** — 按依赖排序，别把技术清理 + 关键词规划 + 全量生产 + 外链 + 分析堆进同一周：
   - 第 1 周：技术基线与度量
   - 第 2 周：关键词架构与页面映射（用 `vibio-keyword` 做这一步）
   - 第 3-4 周：首批优先页面
   - 第 2 月：深度与 on-page
   - 第 3 月：权重 + 复盘节奏
   按工时（精简 3-5h / 中等 6-10h / 强 10h+）调整产出量。

5. **定节奏** — 每周执行块（内容 / 数据与权重 / 技术与内链）+ 每月深度复盘 + 6 个月重置触发条件。

6. **定追踪** — 内容追踪 / 关键词追踪（含验证列与级联列）/ 外链追踪三张表起步，内链追踪在首次内链审计时补建（技术问题日志仅在技术工作量足够时加）。这些表**用 `Write` 落盘到 `.vibio/trackers/`**（按 `vibio-memory` 的 state-templates 建骨架），不是只在回复里生成的文字。

7. **输出 + 写回记忆** — 用 `references/delivery-template.md` 的结构输出，结尾永远给「下一步 3 个动作」。同时用 `Write`/`Edit` 把诊断、主瓶颈、90 天目标、路线图阶段写进 `.vibio/project.md`，建好 tracker 骨架——下次进来才能延续。

## Vibio 默认客户偏向（B2B / 外贸制造）

Vibio 核心客户是出口制造 / B2B 站点，规划时偏向：服务与用例页 → 仅在有真实差异化时做行业页 → 对比/替代/FAQ 资产 → 证据充分的信任页 → 买家教育内容 → 页面质量到位后再做权重。

---

## 按类分支（速查）

- **新站**：技术配置先行 → 长尾定位 → 首批 10-20 个优质页 → 先做商业基础页再铺博客 → 延后外链。
- **存量弱站**：修定位/层级 → 重写弱核心页 → 补缺失的商业页 → 理顺内链。
- **存量内容站**：内容审计 → 合并/重定向/重写 → 刷新排名 11-20 的页 → 修 CTR → 升级内链。
- **电商**：品类/集合页定位 → 产品/集合 schema → 抓取与分面导航控制 → 评价信号 → 品类支撑内容。
- **服务/B2B（Vibio 默认）**：服务与用例页 → 有差异化才做行业页 → 对比/替代/FAQ 资产 → 信任页 → 买家教育 → 后置权重。

## 该跑哪些专家工具

规划里要点名具体的工具，常见组合：
- **关键词研究 / 该做哪些词 / 目标市场验证 / 关键词→页面映射 → `vibio-keyword`**（第 2 周"关键词架构"阶段的主力，可独立触发）
- **内链架构 / 外链获取 / 第 3 月权重启动 → `vibio-link`**（内链 pass 随时可跑；外链按启动闸门）
- B2B 策略 → `b2b-seo`
- 每页内容大纲（含字数）→ `seo-content-brief`
- 内容架构 / 主题簇 → `seo-cluster`
- 站点上线且已验证后的数据 → `seo-google`（GSC / GA4 / CrUX）
- 程序化批量页 → `seo-programmatic`

## 下一步路由（计划生成后）

| 触发条件 | 推荐 |
|---------|------|
| 计划里发现代码库存在具体技术/标记问题 | 「路线图已就绪。第 1 周的技术基线项可以现在就动手——跑 `vibio-fix` 直接改。」 |
| 还没真正看过站点现状，计划基于假设 | 「这份计划基于推断。建议先跑 `vibio-audit` 核实现状，再据实修正优先级。」 |
| 用户要继续推进某个已存在的项目 | 「按延续规则：先定位当前阶段和已完成项，只输出下一段操作切片，不重启整个计划。」 |

---

## 不要做

- 不给没有排序、没有修复动作的泛泛「最佳实践」清单。
- 不把高难度大词作为低权重站的开局动作。
- 在可抓取性、意图对齐、内容质量解决前，不推进阶段性高级战术。
- 在有值得被链接的页面之前，不前置针对页面的外链建设（判据：`vibio-link` 的启动闸门——≥1 个链接资产或 ≥5 个深度页；目录/协会等品牌级基础链接不受此限，上线即可做）。
- 不承诺排名或快速见效。
