# 写作手册 — 从查询证据到可发布页面

WRITE 模式把经验证的买家任务变成可发布页面。它服务文章、产品/服务页、应用页、对比、案例和买家教育，不以固定字数、关键词密度、盲审分数或“AI 可摘引格式”代替质量。

## 一、生产流水线

```text
0. 续接项目与页面任务
1. 验证目标市场查询族、意图和页型
2. 采样当前 SERP 与竞争页面
3. 收集项目一手材料和外部证据
4. 定义信息增益与页面结构
5. 起草并完成 on-page / 内链 / 资产规格
6. 对抗审查硬闸门
7. 目标语言本地化
8. 交付、上线 handoff 与复盘设计
```

外部 `seo-*` 能力均为可选；不可用时用公开 SERP、项目资料和用户导出降级，并标明市场、日期和未验证项。

## 二、续接与任务定义

读取 `.vibio/project.md`、keyword/content tracker、changelog 和目标页面。定义：

```text
Target market / language:
Reader / buying role:
Task and current situation:
Page type and owner URL:
Qualified next action:
Available first-party evidence:
Publication constraints:
```

用户只给 URL/业务而未给查询时，先用 `vibio-keyword` / `keyword-validation.md` 找真实买家语言。不要自动把英语市场设为某国，也不要用“量/难度比”替代企业可服务性和商业选择。

## 三、查询与 SERP 验证

查询族必须通过 `keyword-validation.md`：目标市场表达、SERP 人群/页型、地区需求、搜索者身份和商业路径。第三方搜索量/KD 是供应商估算，保留工具、国家、语言和日期。

对目标地区 SERP 做有界样本，记录：

- 主要任务、页型和混合意图；
- 前排页面的承诺、证据、产品能力、工具与体验；
- 用户仍未完成的问题和过时/含糊之处；
- 企业能真实提供的独特信息。

字数、H2 数和 Schema 数可描述页面，但不能证明内容优劣或可击败性。具体拆解见 `competitor-teardown.md`。

## 四、证据账本与信息增益

先收集产品/销售/工程/客服/测试/CRM/RFQ/案例/认证等项目事实，再核对外部标准、法规、统计和技术结论。使用 `sourcing-and-eeat.md` 和 `evidence-policy.md`：

```markdown
| Claim | Evidence level | Source / owner | Date |
| Exact support | Scope / limitation | Public? | Status |
```

公共论断链接原始来源；内部事实标责任人、时间和公开边界。不可验证的具体数字或经验不得写入。

信息增益必须可指出，例如：

- 产品真实规格、限制、交付或兼容条件；
- 一方测试方法、样本、结果与局限；
- 失败模式、排错路径和不适用场景；
- 案例、决策树、计算器、模板、图像或可下载资产；
- 目标市场特有的法规、单位和采购证据。

不要求机械覆盖前排页面所有小节。保留完成用户任务所需的共识信息，并把资源集中在关键差异。

## 五、页面结构

按任务顺序设计，而非关键词布点：

- 需要快速决定时先给结论与条件；复杂问题先给必要上下文；
- 一个主要部分解决一个清楚的子任务；
- 表格、列表、规格和图像只在更易比较/操作时使用；
- 真实例子、来源和限制放在它们支持的论断附近；
- CTA 承接当前任务，不能突然硬推与意图不匹配的动作；
- FAQ/步骤内容只按用户价值决定，不为已停用的 Google FAQPage/HowTo 富结果添加标记。

`seo-content-brief` 可输出详细大纲，但字数只能是编辑预算，不能要求“略超竞品中位数”。

## 六、起草规则

### 事实与专业性

- 每个高影响专业论断有相应证据或明确项目边界；
- 不制造作者资历、客户结果、标准符合性或测试；
- 说明适用条件、样本、单位、日期和不确定性；
- E-E-A-T 是质量自评框架，不是可添加的标签或单一排名分数。

### 语言与可读性

- 使用目标市场买家自然语言，但不机械重复主词；
- 删除空泛开头、模糊归因和不增加信息的句子；
- 段落和句子长度服从内容复杂度，不设固定 2-4 句或 40-60 字；
- 保留对 B2B 决策真正有用的规格、表格和步骤。

### 转化

主内容若希望被搜索系统理解，应让 Googlebot 合规抓取、渲染和索引。订阅/付费墙内容按 Google 当前 paywalled content 规范标记；下载物和用户访问策略由价值交换、隐私、销售流程和实验决定，不用“AI 引用靠开放正文”作为万能理由。CTA、信任和表单见 `conversion-playbook.md`。

### AI 搜索

Google AI 功能不要求 answer-first、固定切块、特殊 Schema 或 `llms.txt`。写清楚、原创、有证据并完成用户任务即可。其他平台只有在官方文档或有边界实验支持时才做专用适配。

## 七、On-page 与发布规格

- **Title / H1**：准确、独特、与可见页面承诺一致；无固定字符数，多个 H1 不自动构成索引阻断。
- **Description**：准确概括价值，Google 可能按查询生成其他 snippet；无固定字符数或采用保证。
- **URL**：稳定、可读、符合站点既有语言与路由规则；不强制中文内容使用拼音/英文 slug。
- **Schema**：仅添加当前 Google 支持、与可见内容一致且适合页型的类型；它提供机器可读线索并可创造功能资格，但不保证排名或生成式 AI 展示。
- **内链**：链向真正帮助任务的页面，并从至少一个相关可抓取页面建立入链。donor 数量和锚由信息架构决定。
- **出站来源**：链接支撑具体论断的原始来源。
- **图片/视频**：展示真实产品、过程、结果或解释；alt 描述图像在当前语境中的用途，控制加载影响。
- **社交预览**：准确代表页面。OG 属分享展示，不是自然排名修复。
- **国际化**：每种语言独立验证查询与文案，按站点架构处理 canonical/hreflang。

技术实现由 `vibio-fix` 和对应 stack adapter 完成。

## 八、对抗审查

按 `adversarial-review.md` 先判硬闸门：

1. 事实与来源支持；
2. 目标市场意图/页型匹配；
3. 可指出的信息增益；
4. 真实经验和企业可兑现性；
5. 语言、任务完成和可发布性；
6. 合规、隐私、内链和资产规格。

可用并行 agent 时把事实、竞争/意图、语言/任务分开复核；否则串行执行。reviewer 必须得到必要的 SERP 摘要和证据账本。分数只用于同版本比较，不能抵消硬失败，也不能证明 SEO 效果。

修订直到硬闸门通过；连续两轮无实质改善时报告缺少的专家确认、一方材料或资产，不强制固定 agent 数、95 分、五轮。

## 九、本地化

第二语言采用转创作而非逐句翻译：

- 独立验证查询、意图和 SERP；
- 调整术语、单位、法规、案例、CTA 与文化语境；
- 事实含义保持一致，来源适用范围重新核对；
- 让目标语言专业人员或合适 reviewer 检查关键页面。

## 十、交付与复盘

交付：正文、front matter、evidence ledger、来源、内链出入计划、媒体/下载物规格、限制和复盘指标。可编辑环境中落地后验证 HTTP 与渲染产物；URL-only 给精确 handoff。

通过自然发现路径和 sitemap 帮助发现。普通页面可以在 GSC URL Inspection 中检查状态，但不要把手工 request indexing 当批量发布或排名策略；Google Indexing API 不用于普通页面。

复查时间由内容易变性、法规/产品变化、GSC/业务数据和用户风险决定。不机械季度刷新，不只改日期。发布后按 `review-engine.md` 判断数据何时足以复盘。

## 页面类型提示

- **产品/服务/应用页**：清楚承诺 → 适用对象/条件 → 规格/能力 → 证据/案例 → 选择与限制 → 下一步。
- **选型/对比**：决策场景 → 评价标准 → 可核验比较 → 何时选哪种 → 限制/成本 → 产品路径。
- **买家教育/排错**：问题与结论 → 原因/条件 → 操作/验证 → 常见失败 → 相关产品或支持。
- **案例/研究**：背景 → 方法 → 证据 → 结果与限制 → 可迁移学习 → 下一步。

模板是起点，不强制 FAQ、表格或固定 CTA 位置。

## 不要做

- 不为未验证的直译词或错误 SERP 人群生产页面。
- 不把第三方量/KD 称为真实精确数据，也不自动选择某个英语市场。
- 不承诺“明显更好就会排名”或“长尾更快见效”。
- 不为字数、关键词密度、PAA、fan-out 或 Schema 数量填充页面。
- 不强制 answer-first、固定段落、AI 可摘引度或刷新周期。
- 不发布没有真实入口的孤儿页，但也不凑固定 donor 数。
- 不只改发布日期、批量请求索引或承诺数周到数月排名。
