# 关键词验证协议 — 目标市场真实需求

本协议回答：**目标市场的目标客户会不会以这个词或这组表达完成一个值得服务的搜索任务？** 选词依据来自真实买家语言、目标地区 SERP、可核验需求和业务路径，不来自直译、老板直觉或单一工具分数。

## 一、验证状态

每个候选词族只使用三种状态：

| 状态 | 含义 | 处置 |
|---|---|---|
| `pass` | 目标买家、SERP 意图、需求证据和商业路径相互支持 | 进入词页映射与优先级判断 |
| `conditional` | 商业/意图合理，但母语、地区需求或数据仍不足 | 合入相关页面、小规模测试或等待一方数据 |
| `fail` | 人群、意图、市场、产品能力或商业路径明确错配 | 淘汰并记录原因，防止重复投入 |

“工具显示 0”既不自动通过，也不自动失败；它表示该工具没有测到足够数据。

## 二、五道闸门

### Gate 1：母语与买家表达

从目标市场的一手或原生语料找表达证据：

1. 企业自己的 RFQ、CRM、销售电话、站内搜索和客服记录；
2. 目标市场 GSC 查询与 paid search terms；
3. 目标国本土买家、分销商、行业协会、标准机构、媒体、论坛和竞品页面；
4. 目标地区 SERP、Autocomplete、相关问题和站内搜索联想。

记录原文、来源、国家/语言和日期。Autocomplete/PAA 是发现表达的线索，不是精确搜索量或完整需求证明；结果可能受地区、语言、设备和产品政策影响。

不要只用其他出口商网站互相证明英文措辞。一个高质量一方来源可以比多个低质量复述更强，不设“必须出现两次”的机械门槛。

### Gate 2：SERP 意图与页型

采样目标国家、语言和必要设备下的实时 SERP，记录：

- 主要搜索者和任务；
- 排名前列的页型与页面承诺；
- 商业、信息、本地、购物、视频等结果形态；
- 结果是否混合多个意图；
- 计划页面能否以真实能力提供更合适的结果。

结果以消费者/招聘/学术/DIY 为主，而企业只服务 B2B 采购时通常应失败或拆出更明确修饰词。SERP 是当前意图证据，不是 Google 永久“定性”；记录日期，并在重要决策前重采样。

### Gate 3：地区需求与趋势

搜索量必须与目标国家和语言匹配。按优先级使用：

- 自有 GSC 的真实查询、页面、国家与设备数据；
- Keyword Planner 或 Ads search terms 的目标地区数据；
- 口径明确的第三方搜索量、趋势和 SERP 数据；
- Google Trends 的相对趋势；
- Autocomplete、SERP 专门页面、RFQ/CRM 等定性证据。

记录工具、日期、地区、语言、匹配/聚合方式和品牌过滤。不同工具的数值不可直接混用；Keyword Planner 的数据面向广告规划，Ads competition/CPC 不是自然排名难度。

Google Trends 是抽样、归一化的相对数据；Topic 和 Search term 应根据研究问题选择，不强制只用 Topic。低量 B2B 词可能没有可显示曲线，此时不能由空图推出无需求。

### Gate 4：搜索者身份与排除意图

检查查询本身和 SERP，而不是见到一个否定词就一票否决：

| 可能错配 | 常见线索 |
|---|---|
| 求职 | jobs、salary、careers、resume、intern |
| 学术/学生 | thesis、paper、homework、course |
| DIY/消费 | homemade、at home、craft、single piece |
| 免费软件/资源 | free、open source、template |
| 二手/本地零售 | used、near me、single unit |

例如 `supplier research` 可能是采购任务，`research paper` 通常不是。拿不准时回到 SERP 和买家原文，不靠词表硬判。

### Gate 5：商业路径与可服务性

标记查询在认知、评估、决策和售后中的位置，并回答：

- 该企业能否合法、真实地满足需求？
- 用户完成任务后，合理的下一步是什么？
- 页面应承接到哪个产品、服务、询价、购买、工具或销售动作？
- 内容即使获得流量，是否可能带来合格客户或支持销售？

商业词并非永远优于信息词。高质量标准、计算、排错和选型内容可能直接服务买委会；纯高流量百科内容若没有产品或销售路径，则降级。

## 三、买家语言挖掘

按与业务接近程度优先：

1. **RFQ/CRM/销售原话**：规格、应用、标准、异议、交付和采购角色；
2. **GSC**：已有站点实际获得的查询与页面曝光，注意查询匿名化和行数限制；
3. **paid search terms**：实际触发广告的查询及其线索质量，注意隐私阈值、匹配类型和付费环境偏差；
4. **站内搜索/客服**：用户在已有产品目录中找不到什么；
5. **目标市场原生来源**：协会、标准机构、媒体、分销商、论坛、评测和买家社区；
6. **SERP/Autocomplete/相关问题**：扩展任务、修饰词和页型；
7. **第三方关键词/竞品工具**：补量、趋势和竞争样本，保留供应商口径。

制造业买委会中的工程、质量、采购、运营和财务角色可能使用不同表达。将词族映射到角色和任务，但不要为每个角色变体机械建页。

## 四、地区与语言

- 每种语言独立研究，不把词表逐字翻译；
- US/UK/AU 等市场分别验证拼写、单位、行业术语和 SERP；
- 多语页面的主语言、地区定位、URL、canonical 与 hreflang 必须一致；
- 搜索量工具的国家级数据不能冒充城市级精度；
- 没有目标 locale 的实时 SERP 时标 `locale-unverified`。

## 五、零量与超长尾

工具 0 量时，收集互相独立的证据：

- GSC 曾出现相关查询/曝光；
- RFQ、CRM、站内搜索或销售记录出现该表达；
- 目标地区 SERP 存在专门服务该任务的页面；
- Autocomplete/相关问题出现相同任务；
- 产品目录、标准号、型号或法规证明该需求真实存在。

一条证据通常足以把词保留为 `conditional`，但不足以制造精确需求量。多个相近规格/问题可以合入一个完成任务的页面；只有意图、页型或产品能力明显不同才拆 URL。

对于高客单、低频 B2B 任务，评估合格机会和销售价值，不只看总搜索量。仍然不得用虚构订单金额为词背书。

## 六、AI 搜索时代的处理

Google AI 功能可能使用 query fan-out 探索相关子主题，但 Google 的官方建议仍是基础 SEO、可抓取性、有帮助的原创内容和良好页面体验，不需要专用 AI 改写、特殊 Schema 或 `llms.txt`：

https://developers.google.com/search/docs/fundamentals/ai-optimization-guide

因此：

- 用真实买家任务和相关子问题检查覆盖缺口；
- 让一个页面完整完成同一任务，而不是逐个 fan-out 词建页；
- 不使用未经该项目验证的“AI 引用概率”“零点击折扣”或会话查询倍数；
- AI Overviews/AI Mode、传统 Web 与其他平台的可见性分别记录证据，不互相推断。

## 七、无付费数据时的降级链

| 顺位 | 来源 | 能证明什么 | 不能证明什么 |
|---|---|---|---|
| 1 | GSC | 自有站实际曝光/点击的部分查询 | 完整市场需求、匿名查询明细 |
| 2 | RFQ/CRM/站内搜索 | 真实买家语言和商业价值 | 整体搜索量 |
| 3 | Ads search terms / Keyword Planner | 付费环境下的查询与规划量 | 自然 KD、自然 CTR 或必然 SEO 转化 |
| 4 | 目标地区 SERP | 当前意图、页型和竞争证据 | 精确需求量 |
| 5 | Autocomplete / 相关问题 | 表达和任务发现 | 精确频次、完整查询集 |
| 6 | Trends | 相对趋势和地区对比（有足够数据时） | 绝对搜索量 |
| 7 | 第三方免费数据 | 供应商口径内的辅助比较 | Google 官方指标 |

`allintitle:` 结果数、引号结果数和 `site:` 结果数不作为竞争或需求量指标；这些计数不稳定且并非为此用途设计。

全链不可用时输出 `unverified` 候选与获取数据的下一步，不声称已经验证。

## 八、写回格式

```markdown
| Query family | Market/language | Buyer/task | Intent | Owner page |
| Native evidence | SERP evidence | Demand source | Business path |
| Validation | Priority | Decision note | Checked at |
```

`Decision note` 应能让下一位执行者知道为什么通过/保留/淘汰，例如“US SERP 以工业供应商规格页为主；两条 RFQ 出现 ASTM 型号；第三方量未知”。

## 九、不要做

- 不拿直译或其他出口商的重复措辞当唯一证据。
- 不看全球量替代目标国家数据。
- 不把 Ads CPC/competition、第三方 KD/DR 当 Google 指标。
- 不把工具 0 量自动判为无人搜索，也不自动判为高价值。
- 不把 Autocomplete、PAA 或 `site:`/`allintitle:` 计数当搜索量。
- 不因泛词流量大忽略搜索者身份、SERP 页型和商业路径。
- 不为每个同义词、规格或 AI fan-out 变体建独立页。
- 不在缺数据时给虚构搜索量、排名周期、CTR、转化率或 AI 引用数字。
