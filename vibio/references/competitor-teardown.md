# 逆向 SERP 对手 + 一手素材采集 + 制胜大纲

本文件支撑 WRITE 模式的 Step 3-5。目的只有一个：**让这篇稿子提供 SERP 上现有页面没有的增量（information gain）**，而不是把对手内容堆得更长。

---

## 一、逆向 SERP 头部对手（Step 3）

### 抓谁
在目标国家、语言、设备与日期下抽取足以判断意图和主要页型的自然结果样本；样本量由结果多样性、查询风险和可用能力决定。跳过与研究目标无关的广告和纯导航结果。若出现 AI Overview 或精选摘要，只把来源、可核验论断和页面任务记作带日期的观察，不能把表面格式当成需要模仿的排名配方。

抓取方式：优先路由 `seo-dataforseo` 拉 SERP 数据；不可用时 `WebFetch` 逐个 URL 拉正文。被 WAF 挡时换浏览器 UA。

### 逐篇拆什么
对每个对手页，记录这几列，**别只看字数**：

| 维度 | 看什么 |
|---|---|
| 主导意图与形态 | 指南 / 对比 / 选型 / 产品页——和 Step 1 判的意图对不对得上 |
| 覆盖的子主题 | 它回答了哪些子问题（用 H2/H3 抽） |
| 深度 | 每个子主题是一句带过还是真讲透 |
| 数据与证据 | 用了哪些数字/统计/案例？有没有标来源？来源新不新？ |
| 独家角度 | 有没有一手经验、原创对比表、计算器、原始数据 |
| 结构与可读性 | 是否让目标用户容易完成任务；表格、规格或步骤是否真的提高理解 |
| 明显短板 | 过时？泛泛而谈？答非所问？漏了关键子问题？ |

### 产出：「覆盖 vs 缺口」表
把所有对手横向合并成一张表——这是制胜大纲的依据：

```
子主题 / 买家问题        | 对手A | 对手B | 对手C | 缺口判定
-----------------------|-------|-------|-------|------------------
什么是 X / 定义           |  ✓深  |  ✓浅  |  ✓深  | 已饱和，简洁带过即可
X vs Y 选型对比          |  ✗    |  ✓浅  |  ✗    | ★缺口：做一张全维度对比表
具体规格/参数门槛         |  ✓浅  |  ✗    |  ✗    | ★缺口：给真实参数+来源
真实应用案例/失败教训     |  ✗    |  ✗    |  ✗    | ★★最大缺口：上一手素材
价格/MOQ/交期（B2B）     |  ✗    |  ✗    |  ✗    | ★缺口：买家最关心，全员没答
数据时效                 | 2019  | 无    | 2021  | ★缺口：用最新数据并标年份
```

打 ★ 只表示候选差异点，不自动表示需求或价值。竞品集体留白可能意味着买家问题无人满足，也可能意味着没有需求、无法公开、页型不适合或信息不可信。必须再用 RFQ/CRM、GSC/广告搜索词、销售访谈或目标市场 SERP 证据验证，并确认企业能够真实回答。

## 二、一手素材 intake（Step 4）

对手页面没有的一手材料可能提高事实可信度、用户价值和被引用理由，但不保证排名或 AI 引用。写之前主动询问最能改变页面决策、且只有内部人员能可靠回答的问题；问题数量由证据缺口决定。按业务类型选问：

### B2B / 外贸制造（默认）
- 这个料/产品在真实应用里最常见的坑或失效模式是什么？（客户踩过的）
- 和最常被拿来对比的替代材料，你们实测的关键差异是什么（强度/重量/成本/工艺）？
- 客户询盘时最常问、但网上查不到准确答案的规格或问题是哪几个？
- 有没有具体的工厂能力数据可公开：产能、公差、QC 通过率、认证、交期？
- 有没有一个能讲的真实应用/定制案例（行业+解决了什么问题，可脱敏）？
- 退货/投诉里暴露过哪些「买家选错型号」的典型情况？

### 通用信息站 / 博客
- 你（或团队）在这个主题上有什么第一手经验、踩过的坑、反直觉的结论？
- 有没有自己跑过的测试、统计、截图、原始数据可以放进去？
- 你的读者最常误解这个主题的哪一点？

**用法**：把回答直接编进正文作为独家段落，并在证据表里把可公开的内部数据也登记为来源（注明「来自 <公司> 内部数据/案例」）。一手经验是 E-E-A-T 第一个 E（Experience）的直接证据。

如果用户暂时给不出一手素材：明确告知「这篇会退化成二手综述，竞争力受限」，建议至少补一个真实案例或一组内部数据再发。

## 三、制胜大纲组装（Step 5）

把三样东西拼成大纲：

1. **必答骨架** = 对手集体覆盖、且 SERP 验证过买家要看的子主题（覆盖到，但不啰嗦，已饱和的简洁带过）。
2. **缺口补强** = 「覆盖 vs 缺口」表里打 ★ 的格子，每个做成一个有实质深度的小节。
3. **独家注入** = 一手素材 → 真实案例小节、原创对比表、内部数据图。

排序按意图与真实买家任务。**每个要给数据的位置标记 `[需证据]`**，交给证据表逐条核实。大纲成型后自检：页面是否有一个可明确指出、对目标用户重要且企业能证明的增量贡献；没有就回到需求与证据阶段，不用固定差异数量凑门槛。
