# B2B SEO 询盘转化手册

最近核验：2026-07-11。

SEO 的业务价值来自合格需求，而不是排名本身。“有流量没询盘”必须先分清流量错配、页面承诺、信任、交互摩擦、渠道/响应和测量问题，不能套通用转化率或 uplift 数字。

## 1. 诊断顺序

| 层 | 要回答的问题 | 主要证据 |
|---|---|---|
| 流量质量 | 国家、语言、query intent 和买家身份对吗 | GSC、landing page、paid search terms、CRM 淘汰原因 |
| 页面/Offer | 页面是否完成该阶段任务并给出合适下一步 | 页面走查、SERP/买家任务、CTA event |
| 信任 | 采购方能否核验企业、产品、标准、交付和责任 | 实际认证、报告、案例、政策、联系人 |
| 交互摩擦 | 表单/上传/移动端/错误是否阻断完成 | Funnel、error logs、录像/热图、可用性测试 |
| 渠道/响应 | 目标市场渠道是否有人按承诺响应 | CRM timestamps、SLA、渠道 outcome |
| 测量 | 是否追到合格线索、商机和成交 | GA4/GTM、CRM lifecycle、coverage audit |

没有数据时可做页面/表单/信任的人工走查，但必须标为待验证，并把必要埋点列入下一步。不能用“低于 1%”等通用线直接诊断。

## 2. CTA 与页面任务

一个页面应有清楚的主任务和相匹配的下一步：

| 买家阶段 | 页面任务 | CTA 例子 |
|---|---|---|
| 认知 | 理解问题、标准或方案 | 查看相关指南、验证适用性 |
| 评估 | 比较、计算、检查规格 | 获取规格/CAD、咨询工程师、查看测试方法 |
| 决策 | 报价、样品、配置、采购 | Request a Quote、Request Sample、提交 RFQ |

规则：

- CTA 文案准确说明点击后的结果，目标页必须兑现。
- 多角色/路径只有真实存在时才分流，避免选择过载。
- 移动端按钮、键盘焦点、对比度和触控区域按无障碍/设计规范实现。
- Sticky CTA、按钮颜色和位置是实验变量，不承诺固定 uplift。
- 页面可以有辅助 CTA，但必须有明确主次。

## 3. RFQ 表单

字段由“销售完成下一步真正需要什么”决定，不使用 3-5 或 7+ 的普遍阈值。

常见字段池：产品/零件、应用、材料/牌号、数量、图纸/CAD、标准/公差、交期、市场、联系人。按产品和买家阶段选择最小必要集合；可在提交后或销售跟进中补充非必要信息。

实现要求：

- 清楚说明为什么需要敏感/技术信息及允许格式。
- 上传失败、验证错误和提交成功状态可见、可访问、可恢复。
- 不强制无必要的注册。
- 多步还是单步由可用性测试和完成率/线索质量决定。
- 防垃圾措施不能让真实海外买家无法提交。
- 隐藏/服务器字段记录 consent 合规的 landing page、UTM、referrer 和实验 variant。

同时衡量完成率、合格率、销售可处理性和错误率，不能只优化表单数量。

## 4. 可核验信任

- 认证：展示真实适用范围、编号、机构和有效期；能链接官方记录时提供链接。
- 工厂/流程：使用真实、经授权的图片/视频和设备/流程说明。
- 测试/质量：说明标准、方法、样本条件和报告边界。
- 案例：给出约束、基线、动作、结果测量和可迁移限制；脱敏不等于空泛。
- 交期/MOQ/响应：只承诺运营能履行的范围，并说明条件。
- 第三方验证：区分认证、审计、平台会员和普通徽章，不能混用。
- 企业身份：合法名称、地址、联系人、隐私/条款/退换或服务政策一致。

信任元素必须放在用户做决定的位置，而不是堆满首页或页脚。无法证实的徽章和假客户 logo 会反噬。

## 5. 联系渠道与响应

渠道选择来自目标市场实际询盘与 CRM 结果，而不是地区刻板表。表单、邮件、电话、WhatsApp、微信、聊天等每个公开渠道都需要：

- 明确负责人与服务时段。
- 可履行的响应 SLA。
- 预填的产品/页面上下文。
- 独立 tracking event 和 CRM source 字段。
- 失败/无人值守时的替代路径。

不能用“30 分钟以上就撤 WhatsApp”或“live chat 必须 1 分钟”等无项目证据阈值。用自身响应时间、合格率和成交 cohort 决定。

## 6. 内容门控

希望参与搜索的主内容必须能被 Googlebot 抓取、渲染和索引。完全门控会阻断门后内容，但合规订阅/付费墙页面可以按 Google 的 paywalled content 规范标记；是否向所有用户免费开放仍由商业、隐私和实验决定。没有可靠的通用“-94% AI 引用”或“-82% 自然流量”公式。

决策原则：

- 支撑搜索意图的正文、规格解释、选型和关键事实应让 Googlebot 获得合规可索引版本；若对用户订阅/付费，明确标记门控部分并避免 cloaking。
- CAD、定制配置、报价、可下载模板或需要许可/隐私控制的伴生资产可门控。
- 案例/白皮书可采用“公开摘要与关键发现 + 下载完整资产”，前提是公开页自身有独立价值。
- Gate 只收集完成交付或跟进所需的最小信息，并说明用途。

通过 controlled experiment 比较合格线索、用户完成和搜索表现，而不是只比较邮箱数量。

## 7. 测量与 CRM 闭环

GA4 推荐线索生命周期事件包括：

- `generate_lead`
- `qualify_lead`
- `working_lead`
- `close_convert_lead`
- `disqualify_lead`

来源：https://developers.google.com/analytics/devguides/collection/ga4/reference/events

实施：

1. 验证表单成功事件来自真实成功响应，不是按钮点击。
2. 邮件、电话、WhatsApp、下载等使用清楚的独立事件名，不把全部标成等价 Key Event。
3. 在 consent 与隐私边界内把 landing page、campaign/UTM、实验 variant 带入 CRM。
4. 记录合格、跟进、商机、成交和淘汰原因。
5. 使用 landing page/意图簇/市场 cohort 连接 GSC、站内行为和 CRM。

GSC query 不能与 GA4 用户/收入做确定性 join；详细边界见 `roi-attribution.md`。

## 8. 实验

可测试：CTA 承诺、表单字段/步骤、信任证据位置、渠道路径、公开/门控结构和响应流程。预先定义主指标、合格率 guardrail、样本/窗口和停止条件，遵循 `seo-experimentation.md`。

不要同时改流量来源、Offer、CTA、表单和销售响应后把结果归因给一个元素。

## 不要做

- 不在流量和意图错配时只改按钮。
- 不以通用转化率、字段数、响应分钟或 uplift 百分比替代目标企业数据。
- 不强制无必要注册，不用假紧迫感、假库存、假案例或假徽章。
- 不门控页面完成搜索任务所必需的主内容。
- 不发布无人负责的联系渠道或无法履行的 SLA。
- 不把原始 form submit 当作最终 SEO 价值；至少追到合格线索。
- 不做 query 级确定性收入归因。
