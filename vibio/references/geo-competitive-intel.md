# AI 搜索竞品情报

最近核验：2026-07-11。

目标不是给竞品编造 GEO 分数，而是发现：哪些实体/页面被展示、它们提供了什么独特证据、哪些真实买家任务仍无人解决，以及我们能否用更好的资产赢得搜索与业务结果。

## 1. 定义样本

固定记录：

- 平台与产品（Google AI Overviews/AI Mode、ChatGPT web search、Perplexity、Bing 等）。
- 国家、语言、设备/登录状态（可控范围内）。
- 查询或买家任务、采集日期。
- 页面/来源是否出现、具体被引用的事实、链接目标。
- 结果的非确定性和个性化限制。

人工抽样只代表观察，不代表完整排名或平台规则。

## 2. 对比维度

| 维度 | 自有站 | 竞品 A | 竞品 B | 证据 |
|---|---|---|---|---|
| 正常 Search 收录/可见性 | | | | GSC/目标 SERP |
| AI 功能/答案中的出现 | | | | 带日期的平台样本 |
| 被采用的一手数据/观点 | | | | 原文与引用片段 |
| 买家任务覆盖 | | | | Query/page map |
| 原创产品/流程媒体 | | | | 页面资产 |
| 真实第三方佐证 | | | | 媒体、标准、伙伴、评论 |
| 商业路径与可验证信任 | | | | 页面/政策/联系流程 |

不要用 `llms.txt`、FAQPage/HowTo、固定段落长度、Wikipedia 是否存在或 schema 数量计算“AI 权威分”。Google 已明确否定这些做法中的多项作为 AI Search 必修技巧。

## 3. 页面拆解

对实际出现的竞品页面记录：

1. 它完成的用户任务和页面类型。
2. 被展示/引用的具体事实或媒体。
3. 事实来源：一手测试、标准、案例、专家经验还是二手总结。
4. 页面如何被内部链接和外部佐证。
5. 哪些条件、风险、数据或商业下一步仍缺失。
6. 我方是否拥有真实能力做出明显更好的答案。

无法确认平台为何选择某页面时，写“观察到”，不能写成排名因素。

## 4. 三类机会

### 我方缺口

竞品拥有真实资产而我们没有，例如原创测试、产品视频、完整标准对照、专家署名、可验证案例或更清楚的商业路径。决定是补资产、合作获取证据，还是放弃该主题。

### 竞品缺口

竞品回答了主题，但缺少重要条件、失败模式、目标市场标准、真实数据或决策路径。用一手能力补齐，而不是仅改写其结构。

### 集体空白

所有结果都缺少同一真实买家任务。先用 RFQ/CRM/paid search terms 验证需求，再建设可防御资产；不能因为没人做某种 schema 或文件就称为机会。

## 5. 行动类型

- 发布原始测试数据、方法与适用范围。
- 建立真实产品/流程/结果媒体资产。
- 提供条件化的选型、兼容性或 failure-mode 决策框架。
- 修复自有站的收录、渲染、内链或错误事实。
- 获得合法的标准、协会、客户、伙伴或媒体佐证。
- 对平台专项假设设计小范围实验和停止条件。

不得虚构 mentions、制造 Wikipedia/Wikidata 条目、批量复制竞品页面或追求已停用的富结果。

## 6. 节奏

按变化速度、风险、数据成熟度和待决策事项安排复查。重大产品发布、迁移、算法/文档变化、异常流量变化、查询样本漂移，或某项资源分配需要新证据时重新采样；稳定且没有待决策事项时不因月度/季度日历重复工作。

只有会改变页面、资产或资源分配的情报才进入 `.vibio/`。
