# Semantic Content Networks — 语义内容网络

从 hub-and-spoke 主题簇升级为多维度语义内容网络。在网络中，每一个新页面都有明确定义的实体位置、用户旅程角色、内容格式和内链拓扑。

---

## 1. 网络架构：实体 = 节点，关系 = 边

### 实体节点类型
- **核心实体**：你的产品/品牌/公司
- **材料/产品实体**：具体的 SKU、产品线、材料等级
- **属性实体**：规格参数、测试标准、认证
- **应用实体**：行业场景、使用案例
- **问题实体**：买家痛点、常见故障、选型困惑
- **替代实体**：竞争材料/产品、替代方案

### 关系边类型
- `is_a_type_of` — 分类关系（碳纤维布 → 碳纤维材料）
- `used_in` — 应用关系（碳纤维布 → 无人机框架制造）
- `compared_to` — 对比关系（碳纤维布 ↔ 玻纤布）
- `requires` — 依赖关系（航空航天级碳纤维布 → AS9100 认证）
- `replaces` — 替代关系（碳纤维布 → 铝合金板）
- `solves` — 解决关系（碳纤维布 → 结构轻量化）

### 示例：碳纤维领域网络
```
碳纤维材料（核心实体）
├─ is_a_type_of → 碳纤维布
│   ├─ used_in → 无人机框架
│   │   └─ solves → 结构轻量化
│   ├─ used_in → 汽车部件
│   ├─ compared_to → 玻纤布
│   └─ requires → ASTM D3039 测试
├─ is_a_type_of → 碳纤维管
│   ├─ used_in → 机械臂
│   └─ compared_to → 铝合金管
└─ is_a_type_of → 碳纤维板
    ├─ used_in → 医疗设备外壳
    └─ requires → ISO 13485 医疗认证
```

## 2. 用户旅程层

在实体网络上叠加买家旅程阶段：

| 阶段 | 用户需求 | 对应网络节点 | 内容格式 |
|------|---------|------------|---------|
| **认知** | "这是什么？有什么用？" | 实体定义节点 + 应用节点 | 定义页、行业概述、入门指南 |
| **兴趣** | "怎么选？什么最适合我？" | 对比节点 + 属性节点 | 对比页、选型指南、FAQ |
| **评估** | "具体规格是什么？符合我的需求吗？" | 属性节点 + 认证节点 | 规格表、TDS、测试报告、认证说明 |
| **决策** | "怎么买？找谁买？要多少钱？" | 商业节点 + 可信度节点 | 询盘页、案例、RFQ、价格指南 |

每个页面在内容网络中同时占据一个实体位置 **和** 一个旅程阶段——这决定了它的深度和转化意图。

## 3. 格式分配矩阵

不同的实体关系 → 不同的内容格式：

| 实体关系 | 内容格式 | 示例 |
|---------|---------|------|
| 实体定义 | **定义页** — 核心定义 + 分类 + 关联实体 | "What is Carbon Fiber Fabric?" |
| 实体对比 | **对比页** — 全维度对比表 + 决策框架 + 适用场景 | "Carbon Fiber vs Fiberglass: Which to Choose?" |
| 实体应用 | **应用指南** — 按行业展开 + 案例 + 选材理由 | "Carbon Fiber in Drone Manufacturing" |
| 实体规格 | **规格表/数据表** — 参数表 + 测试标准 + 单位 | "Carbon Fiber Fabric Technical Specifications" |
| 实体选择 | **选型指南** — 决策树 + If-Then 推荐逻辑 | "How to Choose Carbon Fiber Fabric: Buyer's Guide" |
| 实体认证 | **合规指南** — 标准解读 + 认证流程 + 供应商筛选 | "Aerospace Carbon Fiber Certification Requirements" |
| 实体问题 | **FAQ/排错指南** — 问题 → 直接答案 → 展开 | "Why Does Carbon Fiber Delaminate? Causes & Fixes" |

## 4. 内链拓扑设计

### 三层链接结构

```
         Hub 页（核心实体）
        /     |     \
   类型A    类型B    类型C  （垂直链接：分类学）
   /  \     /  \     /  \
  应用1  应用2 应用3  对比页  （水平链接：跨引用）
   |      |     |      |
  FAQ   规格   认证   选型    （旅程链接：推进用户）
```

### 链接规则
- **垂直链接**（Hub → 子实体）：每个 Hub 页面至少链向 3-5 个子实体页面
- **水平链接**（同级实体间）：相关实体互相链接，增强主题覆盖信号
- **旅程链接**（推进用户）：认知页 → 评估页 → 决策页，按用户意图递进
- **权威分配**：商业价值越高的页面，获得越多的内链
- **链接新鲜度**：新内容发布后，回头更新已有页面添加指向新内容的链接

## 5. 覆盖度评分

### 三个维度评估网络完整性

**实体覆盖率**：你的网络覆盖了这个领域多少实体？
```
实体覆盖率 = 你已有内容的实体数 / 该领域总实体数
```

**关系覆盖率**：实体间的关键关系有多少被你覆盖了？
```
关系覆盖率 = 你已有内容覆盖的关系边数 / 该领域总关系边数
```

**竞品覆盖叠加**：把你的覆盖图和竞品覆盖图叠加：
- 竞品有、你没有 → **缺口，优先填补**
- 你有、竞品没有 → **差异化壁垒，保持并扩展**
- 都没有 → **蓝海机会，抢占先发优势**

### 格式均衡
网络中是否包含多种内容格式的合理配比？纯指南型网络缺乏商业转化力；纯产品页网络缺乏权威信号。建议配比：40% 教育/认知 + 30% 评估/对比 + 20% 决策/商业 + 10% 深度/专业。

## 6. 网络健康监控

| 指标 | 检查方式 | 频率 |
|------|---------|------|
| **孤儿节点** | 内链数为 0 的页面 → 需要从相关页面添加链接 | 季度 |
| **过链接节点** | 内链数异常高 → 稀释链接价值，需要分散 | 季度 |
| **覆盖缺口** | 高价值实体关系没有对应内容 → 排入 WRITE 队列 | 月 |
| **深度缺口** | 有页面但内容浅（< 800 字、无表格、无来源）→ 标记改进 | 月 |

## 7. 构建顺序

```
Phase 1 — 核心枢纽: 主题支柱页 + 核心实体定义页
Phase 2 — 高价值关系: 对比页 + 应用指南 + 选型框架
Phase 3 — 深度覆盖: 规格表 + 认证指南 + 垂直应用页
Phase 4 — 旅程完成: 案例 + 客户评价 + 询盘/RFQ 页
```

每完成一个 Phase，回头更新上一 Phase 的页面内链 → 网络有机生长，而非一次性建设。

## 8. 与现有模式的集成

- **KEYWORD 模式**：关键词映射时标注每个词在网络中的节点位置和关系类型
- **WRITE 模式**：内容 Brief 中注明该页面在网络中的实体位置、格式类型、应链接到的上下游页面
- **AUDIT 模式**：全站审计时输出内容网络的覆盖度评分和缺口清单
- **seo-cluster**：SERP 重叠聚类是语义网络的输入之一
- **entity-strategy.md**：实体建设策略是语义网络的品牌权威层
