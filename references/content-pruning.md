# Content Pruning & Consolidation Protocol — 内容修剪

Google 的 Helpful Content System 和核心更新越来越奖励删除低质量内容的网站。修剪弱页面可以提升整站质量信号，带动全站排名。这感觉反直觉——但数据反复证明有效。

---

## 1. 内容库存评分

每个页面按 5 个维度评分，取平均（0-10）：

### 评分量表

| 分数 | 流量(90天GSC点击) | 外链(RD数) | 转化(90天) | 新鲜度(距上次更新) | 独特性 |
|------|-----------------|-----------|-----------|-----------------|--------|
| 10 | 200+ | 10+ | 3+ | < 6 个月 | 完全原创 |
| 6 | 51-200 | 4-10 | - | 6-12 个月 | - |
| 4 | 11-50 | 1-3 | - | 1-2 年 | - |
| 3 | - | - | 1-3 | - | 部分重叠 |
| 2 | 1-10 | - | - | - | - |
| 0 | 0 点击 | 0 RD | 0 转化 | >2 年 | 重复/转载 |

### 数据获取程序
- 流量：`seo-google` GSC Search Analytics，过滤到该页面 URL，过去 90 天
- 外链：`seo-backlinks` 查该页面 RD 数
- 转化：GA4 → Landing Page report → 过滤该 URL
- 新鲜度：检查页面发布日期和最后更新日期
- 独特性：人工抽查或使用 `seo-firecrawl` 比较相似页面

---

## 2. 修剪决策矩阵

| 综合分 | 决策 | 动作 |
|--------|------|------|
| 8-10 | **保留** | 继续维护和定期刷新 |
| 5-7 | **改进** | 60 天内做实质性更新，否则降级到合并 |
| 3-4 | **合并** | 与更强的同类页面合并 → 301 重定向 |
| 0-2 | **删除** | 无外链 → 410 Gone；有外链 → 301 到最相关页面 |

### 特殊规则
- **发布不满 12 个月的页面**：不适用 0-2 分删除规则，给更多时间（提升到 3-4 分合并评估）
- **季节性页面**：在淡季时不评估，等到旺季数据完整后再评
- **有 ≥3 个外链的页面**：永远不 410——至少 301 到最相关页面保留外链价值
- **产生过 ≥1 次转化的页面**：至少给改进机会，不直接删除

---

## 3. 合并协议

当两个页面针对重叠关键词时，合并为一个更强的页面：

1. **选存活页**：优先保留外链更多 + 历史排名更好的那个
2. **内容迁移**：把被删页中独特有价值的内容移到存活页（新的小节、独家数据、好案例）
3. **301 重定向**：被删页 → 存活页（精确匹配，不是批量到首页）
4. **更新内链**：全站搜索指向被删页的内部链接，改为指向存活页
5. **更新 sitemap**：移除被删页 URL
6. **GSC 提交**：在 GSC 中请求重新抓取存活页（标记为已更新）

---

## 4. 修剪前基线

修剪的目的是**让剩下的页面排名提升**。必须记录修剪前数据才能证明效果：

- [ ] 记录所有**保留页**的 GSC 数据（过去 28 天：点击、展示、平均排名）
- [ ] 记录全站索引页面总数（GSC → Index → Page Indexing）
- [ ] 记录全站日均抓取量（GSC → Settings → Crawl stats）
- [ ] 将基线数据写入 `.vibio/changelog.md`：`PRUNE BASELINE: [日期] — 全站 X 页面，Y 页面标记为修剪`

---

## 5. 修剪后监控

| 时间点 | 检查项 |
|--------|--------|
| 30 天 | 保留页排名变化？抓取预算是否向重要页面转移？（日均抓取量变化） |
| 60 天 | 合并页面的流量是否恢复？（301 目标页流量 + 被合并关键词排名） |
| 90 天 | 整站质量评估：平均排名是否改善？是否有新的关键词进入前 50？ |

如果 90 天后无改善 → 修剪可能不是瓶颈，回到诊断阶段找根因。

---

## 6. 修剪节奏

| 频率 | 动作 |
|------|------|
| **季度** | 对全站所有页面跑内容库存评分 |
| **每次核心更新后** | 重新评估排名下降的页面 |
| **大内容推送前** | 先修剪，最大化新内容的效果（Google 对新内容的质量评估受全站质量影响） |
| **首次修剪** | 建议先剪最明显的 10-20%（0-1 分页面），观察 60 天效果后再深化 |

---

## 7. 与现有模式的集成

- **REVIEW 模式**：月度复盘中加入修剪候选页面的检查
- **AUDIT 模式**：全站审计时输出修剪建议清单
- **content-decay.md**：衰减检测是修剪库存评分的上游输入
