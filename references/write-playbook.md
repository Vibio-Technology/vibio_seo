# 写作手册（Write Playbook）

WRITE 模式的执行引擎。把一个关键词或选题，变成可直接发布、能在 SERP 上竞争的文章（B2B 行业文、产品页或博客），支持中文和英文。

核心理念：**一篇能排名的文章是从 SERP 逆向推导出来的，不是凭空写出来的。** 先读已经排名的内容，找出空白，收集关键词，再写出明显更好、更完整的东西。重活交给现有的子 skill，本手册是把它们串起来的流水线。

## 生产流水线

按顺序执行。每个阶段为下一阶段供料。只有当本次会话已经拿到数据时，才跳过某阶段。

```
0. 摸底 (Intake)     → 页面类型、选题、市场/语言、目标 URL/代码库
1. 关键词内核        → seo-dataforseo：种子词搜索量、难度、意图、相关词 + 长尾
                       + 目标市场验证（keyword-validation.md 五道闸门，未验证的词不动笔）
2. SERP 侦察         → 读头部词排名前 5-10 的竞品文章
3. 空白 + 切角       → 他们都写了什么、漏了什么、用什么角度赢
4. 集群图谱          → seo-cluster：语义覆盖、同级子话题、内链矩阵（规则见 link-architecture.md）
5. 内容简报          → seo-content-brief：大纲、各段字数、关键词布点、预登记内链
6. 起草              → 按简报写正文（本手册的写作规则）
7. On-page 装配      → 标题、meta、slug、H 标签、schema、内外链、alt
8. 质量闸门          → 5-Agent 盲审（95 分闸门），不达标打回重写，最多 5 轮
9. 本地化            → 如需第二语言版本（转创作，不是翻译）
10. 交付             → 文件/markdown + 发布清单（含内链回填四步）+ 接下来该写的 3 篇
```

如果某个依赖 MCP 的阶段（1、2）不可用，明确说明并降级：用 web search 拿实时 SERP，手动读竞品。绝不能默默跳过关键词依据——要标注出来，并写明所做的假设后再继续。

## 阶段 0a — 自主关键词发现（当用户未指定关键词时）

当用户给的是网站 URL 或代码库，**没有指定种子词**时，在进入阶段 0 之前先跑本阶段，由系统自主推导最值得写的关键词和目标市场。

### 第一步：读懂甲方业务

优先从代码库（产品页、About 页、首页、现有博客）或 URL 提取：
- **核心产品 / 服务** — 具体是什么，材质/规格/用途/应用场景
- **已有内容** — 哪些话题已写，哪些空白
- **显性目标市场** — 网站现有语言、已链接的地区、客户案例地域
- **品牌定位** — OEM/ODM/自有品牌、价格段、认证

可用工具：`seo-firecrawl`（爬取页面结构）、`seo-google`（如有 GSC，拉已获得排名的查询词）、`seo-audit`（快速全站健康 + 现有关键词分布）。

### 第二步：市场判断

根据业务特征推导最优目标市场。通用规则：

| 业务类型 | 优先市场顺序 |
|---|---|
| 外贸制造 / OEM | 英语市场（US > UK > AU）→ 欧陆（DE/NL）→ 中东（AE）|
| 跨境电商 | 视产品类目：US/UK 为主，再看竞品主阵地 |
| SaaS / 数字产品 | US 英语为绝对第一，次选 APAC 英语市场 |
| 国内 B2B | 简体中文，目标百度 + Google 中文 |
| 多语言需求 | 各语言独立调研，不做直译 |

**验证方法**：用 `seo-dataforseo` 对候选市场分别查同一个核心词的搜索量，选择量/难度比最优且与业务匹配的市场。

### 第三步：关键词机会矩阵

路由到 `b2b-seo`（B2B 业务）或 `seo-plan`（其他），结合 `seo-dataforseo`，产出一张关键词机会矩阵：

```
关键词 | 月搜索量 | 难度 | 意图 | 业务相关度 | 优先级
```

**打分逻辑**（各维度评分，综合排序）：
- **目标市场真实性（前置闸门）** — 每个候选词先过 `keyword-validation.md` 五道闸门（母语性/SERP 试金石/目标国量/身份排除/商业路径）；推荐清单里只允许 PASS 词，直译出来的假词在这里就淘汰
- **搜索量 × 业务相关度** — 流量有价值才有意义，量大但不相关的词无用
- **难度 vs 站点权重** — 新站 / 低权重站优先难度 ≤ 30 的词；有权重的站可以打 30-50
- **商业意图** — B2B 场景优先 Commercial + Informational（买家教育）型，不打纯 Navigational
- **长尾优先** — 首批文章优先低难度长尾（volume 100-1k），快速获得排名信号，再渐进打头部词

### 第四步：输出关键词+市场推荐清单

输出格式：

```
推荐目标市场：[语言/地区]
推荐理由：[1-2句，基于业务+SERP数据]

优先关键词（按优先级排序）：
1. [关键词] | 量: XX | 难度: XX | 意图: XX | 推荐页面类型: XX
2. ...
（通常给5-10个，覆盖不同难度梯次）

建议写作顺序：先打 #N（最快见效），同步规划 #M 作为支柱页
```

清单产出后，让用户确认或调整，然后从排名第一的词进入阶段 0 → 阶段 1 正式流水线。**不要等用户逐一批准每篇，给出整体计划后直接开写第一篇。**

## 阶段 0 — 摸底（Intake）

推导，而不是审问。能从会话/代码库里拿到的就自己拿。只确认那些会改变文章走向的事项：

- **页面类型** — `B2B 行业文`（买家教育 / 应用场景 / how-to）、`产品页`、或 `博客`。它决定模板（见"页面类型模板"）。
- **选题 / 种子关键词** — 这个页面要排名的目标词。
- **市场 + 语言** — 主中文、主英文、或双语。注明地区（如 EN-US vs EN-GB、简体 vs 繁体）——它影响拼写、案例和 SERP。
- **目标对象** — 这篇文章服务于哪个站点/品牌（读代码库或 URL，了解语气、产品、内链目标、可互链的已有文章）。
- **主转化** — 页面要驱动什么（询盘、索样、注册、继续阅读）。它决定 CTA。

对 Vibio 核心客户（外贸制造 / B2B），切角默认偏向：买家教育、规格/选型指南、应用场景文、对比/替代品内容、以及信任/背书框架。

## 阶段 1 — 关键词内核（seo-dataforseo）

**先验证，再投产。** 如果种子词不是来自 `.vibio/trackers/keywords.md` 里已标 `pass` 的词（用户现场指定、或本次新发现），先过 `keyword-validation.md` 的五道闸门——尤其是母语性（目标国母语来源里真出现过这个措辞）和 SERP 试金石（目标国 SERP 前 10 的人群 = 目标客户）。验证 FAIL 的词如实告知用户并给出替代词，不为假词写文章；一篇文章几小时的投入，错在选词上就是全损。

然后路由到 `seo-dataforseo`。针对种子词及其市场+语言，拿到：

- **种子词指标** — 月搜索量、关键词难度、搜索意图（信息型 / 商业型 / 交易型 / 导航型）。
- **相关关键词** — 要织进正文以做语义覆盖的一组词。
- **长尾词集** — 搜索量更低、难度更低、意图更强的短语。它们是 H2/H3 和 FAQ 的目标词；低权重站点早期真正能赢的就是这里。
- **问题词** — People-Also-Ask / 问题型关键词 → 变成 FAQ 段落 + schema。

本阶段产出：一个**主关键词**、3-8 个**次要关键词**、一份**长尾词清单**、一份**问题清单**。匹配主导意图——别给交易型查询写指南，也别给信息型查询写销售页（意图与页面类型对不上时，用 `seo-sxo`）。

中英双语产出时：**每种语言分别做关键词调研**。中英文的 SERP、搜索量、意图几乎不会 1:1 对应——把英文关键词直译成中文，往往根本没有中文搜索量。要在每种语言里找到真实的查询词。

## 阶段 2 — SERP 侦察（读竞品）

这是用户特别想要的一步，也是最能拉高质量的一步。对主关键词，拉出排名前 5-10 的页面（用 `seo-dataforseo` 的 SERP，MCP 不可用时用 web search），认真读其中最好的 3-5 篇。对每一篇，记录：

- **格式与页面类型** — 清单体、长指南、工具页、产品页？（匹配主导格式是底线门槛。）
- **深度** — 字数、H2 数量、覆盖的子话题。
- **结构** — 他们先讲什么、论证如何排序。
- **实体与细节** — 他们提到的具名概念、规格、数据点、标准、品牌（Google 奖励话题完整度；缺实体 = 覆盖薄）。
- **E-E-A-T 信号** — 作者、引用来源、原创数据、图片/图表、第一手经验。
- **弱点** — 信息过时、漏掉子话题、没数据、UX 差、没案例、空泛灌水、没回答某个 PAA 问题。**这些空白就是你的突破口。**

当你需要搞清楚这些页面*为什么*能排名、服务的是什么意图/人群时，用 `seo-sxo`（SERP 逆向）——尤其是用户已有页面没能排上去的情况。

把这些整理成一张简短的竞品对照表放进工作笔记，它驱动后面的简报和切角。

## 阶段 3 — 空白 + 切角（赢的策略）

读完竞品后，明确写出三件事，否则不要进入起草：

1. **共识覆盖面** — 排名页几乎都在讲的子话题。这些你**必须**覆盖，否则会被判定为话题不完整。它们是入场券，不是差异化。
2. **空白** — 没人讲好、或根本没讲的东西：缺失的子话题、过时的数据、没回答的问题、没有真实案例、没有规格表、没有决策框架。**这是你超越他们的地方。**
3. **赢的切角** — 一句话说清这篇文章凭什么更好。例如："唯一一篇含真实选型参数表 + 按应用场景拆解的指南"，或"唯一从采购方视角而非卖方视角写的对比"。

切角必须落到**信息增益**（information gain）上：相对已排名内容，提供新的、原创的、更具体的价值。复述竞品已有的内容不会排名。对 B2B 而言，信息增益常来自：真实规格/参数、原厂第一手经验、采购决策框架、行业标准引用、成本/交期/MOQ 等买家真正关心却没人写的细节。

## 阶段 4 — 集群图谱（seo-cluster）

路由到 `seo-cluster`，把这篇文章放进更大的内容架构里：

- **语义覆盖** — 基于 SERP 重叠确认这篇该覆盖哪些关联子话题（不是靠文本相似度猜）。补齐阶段 3 可能漏掉的实体。
- **同级 / 支线话题** — 哪些是该文章内部的小节，哪些该单独成文（避免一篇文章硬塞多个意图）。
- **内链矩阵** — 这篇该链向哪些已有页面、又该被哪些页面链入。Hub-and-spoke 结构：支柱页（pillar）链向所有支线，支线相互链接并回链支柱。

单篇文章可以轻量跳过本阶段，但**成体系产出内容**时它至关重要——它防止关键词自相蚕食（cannibalization），并建立把权重导向钱页（money page）的内链结构。

## 阶段 5 — 内容简报（seo-content-brief）

路由到 `seo-content-brief`，把前面所有阶段合成一份可执行大纲：

- **页面类型模板** — 选对模板（B2B 行业文 / 产品页 / 博客）。
- **逐段字数** — 校准到能赢的深度，通常对标竞品中位数并略微超出（不是无脑堆字数）。
- **关键词布点** — 主词进 H1/首段/结尾；次词分配到各 H2；长尾词进 H3 和 FAQ；问题词进 FAQ 段。
- **关键词密度引导** — 自然、不堆砌。优先语义覆盖（相关实体）而非生硬重复主词。

简报产出后，回头核对阶段 3：**每一个空白都要在大纲里有对应的小节**。简报是你和"已排名内容"的契约——它必须把所有共识覆盖面都包含进来，并明确加进你的差异化角度。

## 阶段 6 — 起草（写作核心）

按简报逐段写。这是 skill 自己做的活，不外包。规则：

**结构与可读性**
- **倒金字塔开头**：第一段直接回答主关键词背后的核心问题，给出 40-60 字的可摘录直接答案（这同时服务于 AI 引用和精选摘要 featured snippet）。不要用"在当今快速发展的……"这类空泛开场。
- 一个 H2 = 一个清晰子话题，措辞贴近真实查询。H3 拆解长尾与具体问题。
- 段落短（2-4 句）。能用表格、清单、对比时就用——B2B 买家在扫读，不在精读。
- 规格、选型标准、对比项一律用**表格**。决策步骤用**编号列表**。
- 每个核心论点能配一个具体例子、数据点或场景就配上。

**E-E-A-T 与信息增益（最关键）**
- 优先**第一手、具体、可验证**的内容：真实规格、参数范围、行业标准编号、典型应用场景、成本/交期/MOQ 等买家关心的细节。
- 引用来源时给出可信依据（标准、机构、原厂数据），不要无出处地断言数字。
- 兑现阶段 3 的切角——每一篇都要带"竞品没有的那一块"。
- 杜绝 AI 灌水腔：删掉"值得注意的是""综上所述""在这个数字时代"这类零信息量短语。每句话要么传递事实，要么推进论证。

**关键词运用**
- 主词自然出现在 H1、首段、至少一个 H2、结尾。
- 次词与长尾词按简报布点，读起来必须自然——宁可漏一个词，也不要读着别扭。
- 用**语义相关实体**（同义词、上位词、关联概念）做覆盖，而不是重复主词。现代 Google 看的是话题完整度，不是关键词密度。

**转化**
- 按阶段 0 的主转化，在文末（视长度也可在中部）放一个贴合上下文的 CTA：询盘、索样、注册、或链到产品页。CTA 层级与措辞按 `conversion-playbook.md` §二的阶梯（信息文用中/软 CTA——下载规格书/继续阅读，别在认知层硬推询盘）。
- CTA 要承接文章逻辑，不要硬广。B2B 场景下，"看具体型号参数 →"比"立即购买"更自然。
- 门控决策按 `conversion-playbook.md` §六：正文永远开放（AI 引用与排名靠它），可下载的伴生资产（规格书/CAD/对比表）才考虑门控。

**长度**
- 对标竞品深度，覆盖完整即止。不为凑字数注水——薄而全 > 长而空。质量闸门（阶段 8）会查灌水。

## 阶段 7 — On-page 装配

正文之外，把可排名要素一次性配齐：

- **Title 标签** — 含主关键词，≤60 字符（中文约 ≤30 字；Google 按约 580-600px 像素宽截断），有点击诱因（年份、数字、收益）。可与 H1 不同。防重写细则（业界大样本研究，方向参考）：别太短（≤20 字符的 title 约半数以上会被 Google 重写）；需要保留的补充语用**圆括号**不用方括号（方括号内容被移除的概率明显更高）；不堆词、不偏离页面主题（否则 Google 直接改用 H1）。
- **Meta description** — 150-160 字符（中文约 ≤80 字），含主词，带利益点和软性行动召唤。预期管理：Google 对 meta description 的改写率约 6-7 成，但高价值页仍值得写——保留时它就是 CTR 杠杆，且 AI 引擎会读它。
- **Slug / URL** — 短、含主词、连字符分隔、英文小写；中文站用对应英文或拼音 slug，别用中文 URL 编码。
- **H 标签层级** — 一个 H1，逻辑清晰的 H2/H3，关键词自然分布。
- **Schema** — 路由到 `seo-schema` 生成 JSON-LD：博客/行业文用 `Article`+`BreadcrumbList`，含 FAQ 段加 `FAQPage`（FAQ 富结果已停显，但结构化 Q&A 对 AI 引用仍有可测的提升），产品页用 `Product`。`Article` 里带作者 `Person`（`name`/`jobTitle`/`sameAs`）——实体化的作者是 EEAT 的技术底座。
- **内链（链出）** — 按阶段 4 的矩阵和 `link-architecture.md` 第三、四节规则：正文每 1000 字 2-5 个上下文内链；链向所属支柱页 + 1 个相关钱页 + 1-2 个同级簇页；锚文本预告目标页内容（禁"点击这里"），指向同一页的锚在不同文章间轮换措辞。
- **内链（链入预告）** — 从阶段 4/5 的简报里确认"发布时哪 2-3 个旧页要插链进来"（`link-architecture.md` 第五节发布协议），把 donor 页和插链位置写进交付清单——这一步留到阶段 10 执行，但现在就要定好。
- **出站链接** — 引用权威来源（标准机构、行业报告）以强化信任；证据表（阶段 6 依据）里的来源 URL 就是出站链接清单。
- **图片** — 配图需求交给 `seo-images`（alt、尺寸、格式、CLS）；需要生成 OG/封面图时用 `seo-image-gen`。每张图写含关键词但自然的 alt。**原创图像（实拍/自制图表）优先于图库**——原创影像是 Google 明确的一手经验信号。
- **视频（如有）** — 按 `video-seo.md` §三：辅助视频嵌进 Article schema（不加独立 VideoObject）、facade 懒加载（CWV 红线）、页面放全文 transcript。选型/how-to 文天然适合配套视频——脚本直接从文章改。
- **OG / 社交预览** — 确认有合规的分享图（横图 1200×630）；缺图时走 `nextjs-playbook.md` 的 OG 修复 + `seo-image-gen`。

## 阶段 8 — 质量闸门：5-Agent 盲审循环（95 分闸门）

**不达标不交付。这是硬闸门，不是建议。** 完整协议见 `adversarial-review.md`。

流程：
1. **发起盲审** — 同时启动 5 个独立 Agent，每个只拿到文章正文 + 自己维度的评分标准，彼此不知道对方存在，也不知道关键词/竞品/大纲/客户（零上下文污染）：
   - Agent 1：信息增益与竞争力（0-20）
   - Agent 2：E-E-A-T 与可信度（0-20）
   - Agent 3：可读性、结构与 AI 可摘引度（0-20）
   - Agent 4：技术准确性与来源质量（0-20）
   - Agent 5：语言质量与去 AI 痕迹（0-20）
2. **判定** — 平均分 ≥ 95 **且**所有单维 ≥ 15 → 通过；否则打回。
3. **修订重审** — 汇总所有 Agent 的具体扣分点，逐条改，重新发起盲审（全新 Agent，无上轮记忆）。
4. **上限 5 轮** — 5 轮仍不过 → 如实报告未通过维度，建议补一手素材或换角度。

盲审 Agent 内部会调用 `seo-content`（E-E-A-T/薄内容）和 `seo-geo`（AI 可摘引度）做专项判断，但最终裁决由 5 维盲审汇总，而不是单个 `seo-content` 打分。

**不要把这步简化成一次 checklist 自检——必须真正启动多个独立 Agent 盲审。**

## 阶段 9 — 本地化（如需第二语言）

要双语时，对第二语言**转创作（transcreate），而不是翻译**：

- 用**阶段 1 中那门语言自己的关键词调研**——不要把第一语言的关键词直译过去。真实查询词常常完全不同。
- 调整案例、计量单位、行业标准、合规引用，贴合目标市场。
- 保持事实与结构一致，但措辞要对母语者自然。
- 多语言站点要配 hreflang——交给 `seo-hreflang` 校验语言/地区标注与对应关系。
- 中→英或英→中时，特别检查：习语、产品术语、以及"在当今"式 AI 直译腔（两种语言都要清除）。

## 阶段 10 — 交付

- 按目标格式输出文章（代码库里的 markdown/MDX，或独立 markdown 文件）。若是 Next.js 项目，按其内容约定（content collection、MDX 等）放置，并跑 build/lint 验证。
- **执行内链发布协议**（`link-architecture.md` 第五节，可编辑站点当场做，URL-only 给出精确清单）：① 在 2-3 个相关旧页正文插入指向新页的上下文链接（donor 页在阶段 7 已定）；② 确认新页链出（支柱 + 钱页 + 同级）已在正文里；③ 所属支柱页加上指向新页的链接；④ 插链动作记入 `.vibio/changelog.md`。**没有入链的新文章是孤儿页，写得再好也打对折。**
- 附一份**发布清单**：title/meta 已填、slug 已定、schema 已加、内链链出已接、**内链回填已做（或列明待办）**、出站来源已链、图片含 alt、OG 图已设、hreflang（如适用）已配。
- 提醒：上线后在 Search Console 提交 URL 以加快收录；新内容自然排名通常要数周到数月。
- 给出**接下来该写的 3 篇**——基于阶段 4 的集群图谱，让内容成体系滚动，而不是孤篇。
- **预排刷新**：内容发布不是终点——AI 引擎显著偏好更新鲜的内容（AI 引用内容平均比传统 top-10 新约 26%），季度刷新节奏优于年度。把这篇的复查时间写进 `.vibio/trackers/content.md` 备注（衰减检测见 `content-decay.md`）。

## 页面类型模板（快速参照）

- **B2B 行业文 / 买家教育** — 开头给直接答案 → 背景/为什么重要 → 核心子话题（选型标准、规格、应用场景，多用表格）→ 常见误区 → FAQ → 链到相关产品页的 CTA。意图：信息型，但承接到商业意图。
- **产品页** — 价值主张首屏可见 → 关键规格表 → 应用场景/适用对象 → 差异化与对比 → 信任信号（认证、案例、MOQ/交期）→ FAQ → 强 CTA（询盘/索样）。用 `Product` schema。意图：交易/商业型。需要对比页时配合 `seo-competitor-pages`。
- **博客** — 钩子 + 直接答案 → 分点展开 → 例子/数据/可视化 → 可操作的收尾 → 软 CTA / 继续阅读。意图：信息型，建话题权威、喂内链。

## 常见错误

- 给未经目标市场验证的关键词写文章——直译假词上的投入是全损（先过 `keyword-validation.md`）。
- 跳过 SERP 侦察直接写——闭门造车出不来能排名的文章。
- 复述竞品已有内容而无信息增益——不会排名。
- 关键词堆砌——优先自然的语义覆盖。
- 把英文关键词直译当中文关键词用（或反向）——查不到真实搜索量。
- 一篇文章硬塞多个搜索意图——拆成多篇（见 `seo-cluster`）。
- 跳过质量闸门——薄内容会拖累整站。
- 发布后不做内链回填——孤儿文章几乎不排名，发布协议四步是交付的一部分。
- 承诺快速排名——新内容要数周到数月，且取决于站点权重。
