# 来源标注 + E-E-A-T / GEO + 去 AI 痕迹

支撑 WRITE 模式的 Step 6-12。三件事：怎么把数据标到可点击的硬来源、怎么写出 E-E-A-T 强且能被 AI 摘引的稿子、写作时怎么去掉 AI 痕迹而不洗掉专业度。

---

## 一、证据表先行（Step 6）—— 防幻觉来源的核心

**LLM 会幻觉一个看起来合理、其实不存在或数字对不上的 URL。** 所以事实核查与写作必须分离：先核实，再写，正文只引用已核实的条目。

### 流程
1. 从 Step 5 大纲里抽出所有 `[需证据]` 的论断/数据点。
2. 逐条 `WebFetch` 核实：URL 真实可达、原文里确实有这个数字/论断、记下原文措辞和发布年份。
3. 填进证据表，写作时**只能引用表里已核实的行**：

```
| # | 论断/数据点 | 可点击 URL | 核实到的原文数字/原话 | 发布年份 | 来源等级 |
|---|------------|-----------|---------------------|---------|---------|
| 1 | 碳纤维拉伸强度约为钢的5倍 | https://... | "tensile strength ... 5x that of steel" | 2023 | A 标准/官方 |
```

4. 核不到、或只找到二手转述找不到原始出处的 → **不写进正文**，或在文中明确标「估计，依据 X」。绝不把没核实的数字当事实写。

### 来源分级（优先引高等级，时效优先）
- **A 级（最优先）**：行业标准与官方文档（ISO/ASTM/GB、政府机构、标准组织）、学术论文、一手内部数据（注明「来自 <公司> 实测/产能数据」）。
- **B 级**：权威行业报告、头部行业媒体、制造商官方规格书。
- **C 级（尽量少用，且需可点击）**：一般媒体、综述类文章。
- **不可用**：内容农场、无署名聚合页、AI 生成的拼接页、Wikipedia 当终极来源（可顺着它的 references 找到 A 级原始出处再引）。

**时效铁律**：每个数据查发布年份。别拿 2015 的统计当现状描述；行业数据尽量用近 2-3 年的。新旧数据并存时，正文写最新的，必要时注明趋势。

### 正文引用形态
- 内联可点击链接锚在数据本身：`碳纤维拉伸强度约为钢材的 5 倍（[来源](url)，2023）`。
- 文末附**来源清单**（「参考来源」列表），每条带标题+URL+年份。
- 一手内部数据注明来源主体，体现 Experience。

## 二、E-E-A-T 写作配方（Step 9）

让 Google 与 AI 都认这篇是「有经验的专家写的、可信的」：

- **Experience（第一手经验，最稀缺）**：把 Step 4 采来的一手素材写成具体段落——真实案例、踩过的坑、实测差异、内部数据。具体到「我们在 X 应用里实测发现 Y」远胜泛泛而谈。
- **Expertise（专业度）**：用准确的行业术语和参数；给选型逻辑/判断框架，不只罗列事实。
- **Authoritativeness（权威）**：每个专业论断锚到证据表里的硬来源；具体出处替代「专家认为/研究表明」。
- **Trust（可信，最重要）**：数据有源可点、有发布年份、利益相关时透明、不夸大不承诺绝对。

## 三、GEO / answer-first（Step 9）—— 决定能不能被 AI 引用

AI Overviews / ChatGPT / Perplexity 摘引的是「能脱离上下文单独成立的、直接回答问题的段落」。所以：

- **答案前置**：每个 H2 小节开头第一句直接回答这个小节的问题，再展开。整篇开头先用 2-3 句直接回答主关键词背后的核心问题。
- **段落自包含**：每段不依赖上一段也能读懂、能被单独摘出来引用。避免「如上所述」「前面提到」这类跨段依赖。
- **实体与定义明确**：关键术语首次出现给清晰定义；产品/材料/标准用全称。
- **结构化承载事实**：对比、规格、参数、步骤用表格或列表——这既是 B2B 可读性刚需，也是 AI 最容易摘引的结构（这一点与下面 humanizer 的「去列表」规则冲突时，**信息结构优先**）。
- **FAQ 化买家真问题**：把 Step 3 发现的「集体留白」买家问题做成 Q&A 块，天然适配 AI 问答式摘引（落地时配 FAQPage schema）。

## 四、去 AI 痕迹（Step 10 写作时启用 + Step 12 第二轮专查）

基于 Wikipedia「Signs of AI writing」整理的可执行规则。**写作时就当硬约束用，不是最后清洗。** 第二轮审稿专查第一轮遗漏的。

### 内容层（全采纳——这些逼你更具体、更有据，直接提升专业度）
- **意义膨胀**：删「pivotal moment / 关键时刻 / 至关重要的一步」这类空话，换成具体事实。
- **模糊归因**：「Experts believe / 研究表明 / 业内普遍认为」→ 换成证据表里的具体出处。表里没有就别写。
- **肤浅的 -ing 分析**：「symbolizing / reflecting / 体现了 / 彰显着」→ 删，或补真实来源与机制。
- **宣传性语言**：「breathtaking / world-class / 顶尖的 / 卓越的」→ 中性事实描述。
- **套路挑战句**：「尽管面临挑战，X 仍蓬勃发展」→ 换成具体数据或事实。
- **信息缺口免责**：「现有资料有限 / 据可查信息」→ 去找来源，找不到就删掉这个论断。

### 语言层（采纳）
- **AI 词汇**：testament / landscape / showcasing / delve / 赋能 / 助力 / 打造 / 不断 → 换普通词。
- **避免系动词**：「serves as / boasts / 充当着 / 坐拥」→ 直接用 is/has「是/有」。
- **否定平行结构**：「不仅是 X，更是 Y」「It's not just X, it's Y」→ 直接陈述观点。
- **三元列举凑数**：「创新、灵感与洞察」这种为对称硬凑三项 → 用真实的数量，有两点写两点。
- **同义词循环**：同一概念别换着花样称呼，选最清晰的一个反复用。
- **假范围**：「from the Big Bang to dark matter / 从入门到精通」→ 直接列真实主题。
- **无主语被动**：「据估计 / 被广泛采用」→ 点名是谁做的。

### 风格层（采纳，英文排版规则中文按需）
- **行内标题列表**：`**性能：** ...` 这种伪小标题 → 能成段就写成散文。
- **预告式导入**：「让我们深入了解 / Let's dive in」→ 直接开始内容。
- **碎片化标题**：标题后跟一句废话过渡 → 让标题自己说话，直接进正文。
- **权威劝说套语**：「At its core / 从本质上讲 / 归根结底」→ 直接说观点。
- **格言公式**：「对称是信任的语言」这种伪金句 → 换成实际论断。
- **假坦诚开场**：「老实说？这取决于…」→ 去掉套路，直接答。
- **制造戏剧的碎句**：长短句自然混合，不要刻意制造短句节奏。

### 沟通层 + 填充层（采纳）
- 删聊天套话：「希望对你有帮助 / I hope this helps」「问得好 / Great question」。
- **填充短语**：「为了能够 / In order to」→「为了 / To」；「进行优化」→「优化」。
- **过度对冲**：「可能也许大概 / could potentially possibly」→ 一个「可能/may」。
- **泛泛结尾**：「未来一片光明 / The future looks bright」→ 换成具体计划或事实，或直接删掉结尾段。

### B2B 技术结构豁免（不无差别套用上面的洁癖规则）
humanizer 原规则有几条为维基百科散文设计，套到 B2B 技术内容会**降低专业度和 AI 可摘引性**，对承载真实信息的结构豁免：
- **保留规格/参数/对比的粗体与表格**：选型文里规格门槛、参数对比本就该加粗、上表，这是技术可读性刚需，也是 GEO 摘引的最佳结构。
- **保留功能性列表**：规格清单、步骤、对比要点用 bullet/表格是对的，别强行改散文。
- **判据（统一标准）**：删了**不掉信息**的格式 = 为排版而排版，删；删了**会掉信息**的结构 = 承载内容，留。空泛的装饰性 bullet 删，装着真实参数的 bullet 留。

## 五、对抗式审稿清单（Step 12 的评分细则）

**这份清单是 `adversarial-review.md` 里 5 个盲审 Agent 的评分依据细则**——不是替代盲审的自检 checklist。执行时走 `adversarial-review.md` 的 5-Agent 盲审循环（95 分闸门），本清单为每个 Agent 提供具体的扣分判据。下面每条对应到哪个盲审维度：

| 本清单条目 | 对应盲审 Agent |
|-----------|--------------|
| 信息增益、意图匹配 | Agent 1（信息增益与竞争力）|
| 每个数据有源、来源时效、一手经验落地 | Agent 2（EEAT）+ Agent 4（来源质量）|
| answer-first、B2B 结构 | Agent 3（可读性/结构/AI 可摘引）|
| 水段检测、AI 痕迹第二轮 | Agent 5（语言质量/去 AI）|
| 内链 | Agent 3 + On-page 装配复核 |

逐条判据（供盲审 Agent 引用）：

- [ ] **信息增益**：相对头部对手，这篇至少有 3 个它们没有的东西？（独家数据/一手经验/原创对比/更准选型）说不出就回缺口表重写。
- [ ] **每个数据有源**：正文每个具体数字/统计/专业论断都能在证据表里找到对应的已核实条目？有没有漏标、有没有写了表里没有的数？
- [ ] **来源时效**：引用的数据发布年份够新？有没有把旧数据当现状？
- [ ] **水段检测**：逐段问「删了这段会不会丢信息」——不丢就删。
- [ ] **意图匹配**：文章形态答到了 Step 1 定的搜索意图？没有跑偏成另一种形态？
- [ ] **answer-first**：开头和每个 H2 第一句直接回答了问题？段落能被单独摘引？
- [ ] **一手经验落地**：Step 4 采的素材真的写进去了，不是又退回二手综述？
- [ ] **AI 痕迹第二轮**：专查第一轮遗漏的套话、模糊归因、AI 词汇、伪金句。
- [ ] **B2B 结构**：承载信息的表格/参数列表还在，没被洗成扁平散文？
- [ ] **内链**：从 content/keywords tracker 取的目标页内链锚文本自然、相关？密度和链出目标符合 `link-architecture.md` 规范（每 1000 字 2-5 个，链向支柱 + 钱页 + 同级）？donor 回填计划已列进交付清单？
